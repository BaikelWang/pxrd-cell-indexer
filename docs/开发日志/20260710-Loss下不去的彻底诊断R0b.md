# 2026-07-10 — Loss 为什么下不去：彻底诊断（R0b）

> **状态**：✅ 完成（7 组可证伪小实验，结论互相印证）  
> **一句话结论**：**loss 下不去是欠拟合，根因是 RealPXRD 峰-token encoder 无法把「峰位→晶格」这个逆问题编码进表示**——与数据量、模型容量、2θ 取整、回归头都无关。同样 700 条样本，换成「Q=1/d² 直方图 + 普通 MLP」可过拟合到 **angle 0.37° / elem 99%**；而当前 encoder（任何规模）都卡在 **~12° / elem 0%**（等价于预测均值）。  
> **前置**：[`20260710-Raw精度根因诊断R0.md`](20260710-Raw精度根因诊断R0.md)（现象层）、[`20260709-算法现状与理论缺陷分析.md`](20260709-算法现状与理论缺陷分析.md)  
> **脚本**：`scripts/{diagnose_raw_errors,probe_encoder,overfit_capacity,overfit_mlp_hist,analyze_quantization}.py`  
> **产物**：`results/beat_engine/raw_diag/*`

## 0. 问题定义

R0 现象诊断已知：严口径 raw elem 只有 ~6.6%，非立方 ~0.17%，`angle_mae≈9.6°`，75% 角被拉向 90°。  
本轮回答**更本质**的问题：**为什么训练 loss 降到 ~0.25 就不动了？** 是数据不够、模型太小、输入信息不够、目标不可学，还是 encoder 本身的问题？逐一证伪。

关键背景：训练 loss 是 **归一化空间 SmoothL1**。「预测数据集均值」对应 loss ≈ **0.25–0.30**。下面多处出现的 ~0.25 平台 = **模型基本在预测均值、没在用输入**。

## 2. D-A 过拟合测试：连 700 条都记不住

配置：`overfit700_seed42.jsonl`（每晶系 100，共 700），train==valid，关增强、dropout=0、weight_decay=0、80 epoch、encoder 解冻。

| ep | TRAIN loss | TRAIN ang | memo elem | memo ang | memo lenMAE |
|---:|---:|---:|---:|---:|---:|
| 1 | 0.378 | 12.5° | 0.0% | 12.3° | 1.58 Å |
| 20 | 0.271 | 10.9° | 0.0% | 10.8° | 1.13 Å |
| 40 | 0.261 | 10.6° | 0.0% | 10.6° | 1.09 Å |
| 67 | 0.255 | 10.5° | **0.3%** | 10.5° | 1.08 Å |

**曲线在 ~ep30 就基本平**；再训不动。一个能自由记忆的模型对 700 条样本应能把 train loss 压到近 0——这里压不动，说明**不是优化没跑够，而是表示接不住**。

### D-A2 连续 2θ（排除取整）

同 700 集、连续位置编码：loss 卡 **0.32**、ang **11.4°**、elem ~0.1%。比 discrete 还差（连续位置 MLP 随机初始化，需更久，但**同样平台**）。→ **2θ 取整不是 loss 平台的原因**。

### D-A 容量扫（排除「模型太小」）

从头随机初始化，只改 encoder 规模，120 epoch 过拟合 700：

| 预设 | 参数量 | train ang | train len_rel | elem |
|---|---:|---:|---:|---:|
| tiny 32d×2L（=现网） | 165K | 11.7° | 0.197 | 0.0% |
| mid 128d×4L | 764K | 12.2° | 0.189 | 0.1% |
| large 256d×6L | **3.56M** | 12.6° | 0.212 | 0.0% |

**参数量放大 20× 完全无改善**。→ 不是容量问题，是**这种「峰当 token」的编码方式**接不住。

## 3. D-A3 对照：同任务，换 MLP+物理特征 → 秒学

把每个样本的峰表转成物理特征：`Q = 1/d²`（`d = λ/(2 sinθ)`）的 **256-bin 直方图** + 排序后前 24 个 Q + 峰数；喂一个 4 层 MLP（约 0.5M 参数），过拟合同一 700 集：

| ep | loss | ang | len_rel | elem |
|---:|---:|---:|---:|---:|
| 100 | 0.222 | 9.4° | 0.167 | 0.1% |
| 400 | 0.043 | 3.4° | 0.077 | 6.3% |
| 1000 | ~0.007 | ~1.0° | ~0.014 | ~95% |
| **2000** | **0.0035** | **0.37°** | **0.007** | **99.0%** |

**同样的 700 条、同样的标签、同样的 6 维目标**：物理特征 + MLP 把 loss 干到 **0.0035**（比 encoder 的 0.25 平台低 **70×**），angle 到 **0.37°**，几乎完全记住。

> **决定性对照**：任务可学（MLP 0.37°）↔ 当前 encoder 学不动（12°）。差别只在**输入表示 / encoder 架构**。

## 4. D-B 探针：瓶颈在表示，不在回归头

对 B3a 的 encoder 抽 CLS embedding：

| 探针 | angle_mae | elem | 说明 |
|---|---:|---:|---|
| 整模型（MLP head） | 9.6° | 6.6% | 参照 |
| **线性探针**（emb→lattice 最小二乘） | **11.1°** | 0.2% | 线性头几乎追平整模型 |
| 预训练 encoder 线性探针 | 11.5° | ~0% | 同量级 |

线性头 ≈ 非线性 head → **head 不是瓶颈**；表示本身只能到 ~11°。

kNN（valid 样本在 embedding 空间找最近训练样本，比较其晶格）：

| 量 | 值 |
|---|---:|
| 最近邻 max 角度差 中位 | **15.9°** |
| 最近邻 max 边长相对差 中位 | **21.8%** |
| 最近邻落在 3°/5% 的比例 | **19.6%** |

**embedding 相近 ≠ 晶格相近** → encoder 没有把晶格几何编码进表示。

## 5. D-C 信息层：输入信息足够，取整损失很小

2θ 取整到整数度对 d 间距的影响（valid 33934 个峰）：

| 量 | 值 |
|---|---:|
| d 相对误差 中位 / p90 / p99 | **0.48% / 1.2% / 2.6%** |
| d 相对误差 > 5% 的峰 | **0.1%** |

→ 整数度 2θ 引起的 d 误差**中位仅 0.5%**，远小于 5% 边长阈值。**取整不是主要信息损失**（也解释了 D-A2 为何没救）。

输入歧义（按整数-2θ 占位签名找最近邻）：

| 量 | 值 |
|---|---:|
| 高相似对（Jaccard≥0.8）占比 | 6.1% |
| 其中晶格落在 3°/5% 的比例 | **74%** |
| 完全碰撞（整数签名相同）样本 | **32 / 1400** |

→ 峰位近乎相同的样本，晶格也近乎相同；**几乎无「同输入不同标签」的本质歧义**。信息在峰里，是 encoder 没用上。

## 6. D-D 真训练曲线：欠拟合，不是过拟合

100k 训练（baseline / B3a）：

| run | TRAIN loss（末） | VALID loss（末） | 说明 |
|---|---:|---:|---|
| baseline | 0.257 | 0.244 | valid 甚至更低（train 有 dropout） |
| B3a | 0.252 | 0.236 | 同上 |

- **train ≈ valid，无泛化间隙** → 典型**欠拟合**。
- train loss 平台 **~0.25**，与 **700 条过拟合的平台完全一致**。
- 含义：**再多数据也没用**——模型连现有数据都拟合不了。

## 7. 综合根因

```
现象:  raw angle_mae ~10°, 严口径 elem ~6%, 角被拉向 90°(=预测均值)
  │
  ├─ 数据量?      否 (train≈valid, 平台与700一致)          [D-D]
  ├─ 模型容量?    否 (32d→3.5M 全卡 12°)                    [D-A-cap]
  ├─ 2θ 取整?     否 (d 误差中位 0.5%; 连续 2θ 也卡)        [D-C, D-A2]
  ├─ 输入歧义?    否 (高相似峰对 74% 一致; 碰撞 32/1400)     [D-C]
  ├─ 回归头?      否 (线性探针≈整模型)                       [D-B]
  └─ 任务可学?    是 (MLP+Q直方图 → 0.37°/99%)              [D-A3]
        │
        ▼
  根因: RealPXRD 峰-token encoder 无法编码「峰位→晶格」逆问题
        表现为 embedding 不保晶格几何、输出坍缩到均值
```

**机制推断**：晶格逆问题的核心是 `Q = 1/d² ∝ (hkl) 的二次型`，需要模型在峰之间做 `sinθ→d→1/d²` 与跨峰的数论/几何组合。当前 encoder 把 **2θ 当作位置嵌入的整数索引、强度当 token**，在 **32 维、2 层** 的小注意力里，从未显式暴露 Q/d 结构；且 encoder 是为 RealPXRD 的其他目标预训练的。给它显式的 Q=1/d² 坐标（D-A3），任务瞬间可学 → **是「怎么把峰喂给模型」错了，不是模型不够大或数据不够多。**

## 8. 与既有结论的关系（更新）

| 既有说法 | 本轮修正 / 强化 |
|---|---|
| 「几何回归太粗」（现象） | **强化**：根因定位到 encoder 表示，非泛泛而谈 |
| 「2θ 取整是高优先级嫌疑」（Phase3 T5） | **下调**：d 误差中位仅 0.5%，非主因 |
| 「容量偏小可能是瓶颈」（缺陷分析 3.1） | **部分否定**：放大到 3.5M 无改善；问题在**输入表示**而非纯容量 |
| 「点估计范式 / 无 Q 闭环」（缺陷分析 3.2） | **强化**：显式 Q 特征即可学，说明缺的是几何特征通道 |
| 「数据不够，需 scaling」 | **否定（当前架构下）**：欠拟合，scaling 先不会救 raw |

## 9. 对 R1 的直接建议（按杠杆排序）

1. **P0 — 换/增输入几何特征通道**：把 `Q=1/d²`（或 d、sinθ）作为**显式特征**喂给模型（拼进 token、或加一路物理特征分支），而不是只靠整数-2θ 位置嵌入。D-A3 证明这一步就能把可学性从 12° 拉到亚度级。  
2. **P0 — 重估 encoder 架构**：当前 32d/2L 峰-token + L2 归一化 CLS 会坍缩；考虑 (a) 更合适的 set/point encoder，(b) 去掉或放宽 embedding L2 归一化，(c) 直接用 D-A3 的 MLP-on-Q 作为强基线对照全量。  
3. **P1 — 先在小集验证可学性再上量**：任何新表示都先过「700 过拟合」闸门（目标 train elem>80%），过了再 scaling；避免重蹈「欠拟合还加数据」。  
4. **P1 — 保留严口径 + per-CS + pull90 监控**（R0 已建）。  
5. **暂缓**：500k/全量 scaling、loss 微调、Top-K/FOM——在 encoder 表示修好前都被欠拟合封顶。

**R1 建议 Gate**：新表示在 700 集过拟合 train strict elem ≥ 80%（现 0.3%）；然后 100k valid strict elem ≥ 15% 且 `angle_mae` ≤ 6°。
