## 1. D9 损失函数 — 建议

### 1.1 核心建议：**晶系条件化 mask 回归**，而不是无差别回归 6 参数

Chitturi et al. (2021) 的做法是**按晶系训练独立模型**，理由：
> "each crystal system has a different number of independent lattice parameters... it is possible for a crystal to be indexed in more than one crystal system; these types of non-one-to-one tasks are typically more challenging"

我们不必真训练 7 个独立网络（工程成本高），但应保留其 **inductive bias**：
- Head 仍输出 6 个数值 `(a,b,c,α,β,γ)`，但 **loss 只在该晶系的"自由参数"位置计算**：
  - 立方：只有 `a` 参与 loss（`b=c=a`, `α=β=γ=90°` 是确定性填充，不计入 loss，也不需要模型"假装学习"）
  - 四方/六方/三方：`a, c` 参与（`b=a`）
  - 正交：`a, b, c` 参与（角度全部 90°，不计入）
  - 单斜：`a, b, c, β` 参与
  - 三斜：全部 6 个参与
- 好处：
  1. 避免模型在恒定角度（90°/120°）上"刷 0 误差"稀释梯度信号；
  2. **晶系-参数一致性约束是结构性保证的**（PM 提到的"是否加一致性约束"问题不再需要额外惩罚项，mask 本身就是约束）；
  3 与我们已定的 D7（晶系由 primitive 推导）完全兼容。

### 1.2 回归 loss 形式

- **a/b/c（长度）**：跨度大（~3–30+ Å），直接 MSE 会被大晶胞样本主导梯度。建议 **MAPE 或 log 空间 MAE**（`|log a_pred - log a_true|`），与 Chitturi、AIdex 两篇论文报告口径（MAPE<5–10%）一致，也方便未来直接对标 AIdex 的 MAPE<5% 目标；
- **α/β/γ（角度）**：分布是"多峰 + 大量 delta 在 90°/120°"，建议 **Huber/Smooth-L1**（对 90° 附近样本更稳，MSE 对偏离时惩罚过陡）；
- 不建议 sin/cos 编码——我们的角度范围（多在 60°–120°）不会遇到周期性 wrap-around 问题，直接回归数值即可。

### 1.3 晶系分类 + 回归的联合权重：建议用 **uncertainty weighting** 自动学，而不是手调 λ

参考 Kendall et al. 2018：
```
L = (1/2σ_cls²)·CE + logσ_cls + (1/2σ_reg²)·L_reg + logσ_reg
```
`σ_cls, σ_reg` 是可学习标量（用 `log σ` 存储保证正数），随训练自动收敛到合理相对权重。相比人工 grid search 更省成本，也是目前多任务学习的主流做法（CV/气象多任务预测均有沿用）。

**初实验（10k smoke）建议**：先用固定权重（如 1:1）跑通 pipeline，验证 loss 不发散、Top-K 评测脚本可用；跑第二轮再切换到 uncertainty weighting 做对比。

## 2. Top-K 实现方式 — 建议

### 2.1 推荐方案 A：**晶系头天然 Top-K**（低风险、可解释、与传统 indexing 工具输出形态一致）

- Encoder 共享，晶系分类头输出 7 类 softmax 概率；
- **每个晶系配一个独立的 lattice 回归子头**（7 个头，共享 encoder，参数量很小），每个头只输出该晶系的自由参数（见 1.1）；
- 推理时，按晶系分类概率排序，取 **Top-K 个"晶系＋对应回归头输出"组合** 作为候选表；
- 优点：
  - **天然多样性**：候选跨越不同晶系（这正是 McMaille/JADE 等传统工具给出的候选形态——不同晶系/倍胞的候选并列）；
  - **训练稳定**：每个头独立监督（ground truth 晶系对应头才计 loss），**没有 WTA 常见的 head collapse 风险**；
  - 与已定的 lattice match Top-K 评测（`ltol=0.3, atol=10°`）天然对齐：MCL 文献里的 "oracle/hindsight loss"（K 个候选里命中一个即算对）评测思路，与我们已经定的指标完全一致，不需要额外改评测逻辑。

### 2.2 可选叠加：晶系内模糊性 → 小型 WTA 子头（非必需，可后置）

若同一晶系内部仍有模糊性（真实 indexing 常见的"倍胞/子胞混淆"），可以在该晶系头上叠加 2 个子候选，用 **Winner-Takes-All (WTA)** 训练（Guzman-Rivera 2012；Stochastic MCL, Lee 2016）：训练时每个样本只更新损失最小的子头。

⚠️ 已知问题：原始 WTA **对初始化敏感、容易头坍缩**（Annealed MCL, NeurIPS 2024 专门解决此问题）。**建议**：如果做，优先做退火版（aWTA/aMCL），而不是原始 WTA；但鉴于我们已经有晶系这个天然的离散多样性锚点，**这一层可以留到 baseline 跑通之后再加，不阻塞初实验**。

### 2.3 备选方案 B（不推荐做 baseline）：RealPXRD 式重采样

RealPXRD 通过 `num_evals` 次 `model.sample()` 走完整 diffusion 过程拿多样本。这对我们的低维输出（6 参数+晶系，远比全原子结构简单）**杀鸡用牛刀**，训练/推理成本高很多，不建议作为 baseline。

**建议结论**：Top-K 采用 **方案 A**（晶系头自然 Top-K），WTA 子头作为后续增强，不阻塞初实验。

## 3. Encoder 冻结策略 — 建议

参考迁移学习经典实践（ULMFiT 式 gradual unfreezing + discriminative LR）：

- **阶段 1（10k smoke）**：**冻结 encoder**，只训练晶系头 + lattice 头。
  - 目的：快速验证数据管线、loss、Top-K 评测脚本是否 work，排除 encoder 本身干扰；
- **阶段 2（全量 241113 训练）**：解冻，但用 **差异化学习率**（encoder LR ≈ head LR 的 1/10～1/100），或先 freeze 训 N epoch 再解冻。
  - 理由：RealPXRD encoder 是在 **600 万+ 结构**的生成任务上预训练的；我们下游数据量级和任务都比它小/窄很多，大 LR 端到端微调容易把预训练学到的 PXRD↔晶胞几何关联"冲坍"（catastrophic forgetting）；
  - 佐证：RealPXRD-Solver 正式论文强调其 encoder 被设计为 **"universal XRD encoder shared between simulated and experimental domains"**——即它本身被设计成任务无关的通用表征器，这也是为何 decoder 能直接从它的输出里读出 lattice 信息（`lattice_out` 由同一个 embedding 生成）。冻结它做 indexing 任务，先验上是合理的。
- **建议做一次低成本 A/B**：10k smoke 阶段同时跑 `frozen` vs `unfrozen(小LR)` 两组，用 lattice match rate 对比，再决定全量训练策略，不用纯凭经验拍板。

## 4. 额外重要发现（超出原三问，但强烈建议关注）：D11 仪器泛化有了直接证据

RealPXRD-Solver **正式论文**（比我们本地代码库更新的版本）明确指出，他们后来把输入表征从 **2θ-I 换成了 d-I（interplanar spacing—intensity）**，并且做了消融验证：

> "diffraction profiles measured under different conditions (background noise level, peak width/FWHM, 2θ range, and step size) nevertheless collapse to a consistent d-I fingerprint, demonstrating that this fingerprint is largely invariant to experimental settings and sample quality."

这直接对应我们此前 D11 列出的候选方向 **(d) 2θ→d 间距输入**——**这不是一个未经验证的猜测，而是原论文实证有效的主线方案**，且论文 Table S2 显示该表征对 FWHM、2θ 范围、扫描步长都有鲁棒性增强。

**关键点**：
- `d = λ / (2 sin θ)`，只需已知 λ 即可从 2θ 换算，**改动量很小**（只改预处理，把喂给 encoder position embedding 的 `pxrd_x` 从 2θ 值换成 d 值的离散化索引，模型结构基本不用动）；
- 我们本地的 `archive/RealPXRD-Solver` 代码（`BertModel.embed_positions` 对 `pxrd_x.long()` 做 2θ 索引）**看起来是论文里更早的版本**，还没上这个改进；
- **建议**：D11 的优先级可以重新评估——鉴于成本低、文献已验证收益，可以考虑在 **10k smoke 阶段就顺手把 2θ 转成 d** 作为预处理步骤（即便训练数据仍是单一 Cu Kα，用 d 表示也没有坏处，且为后续多 λ/仪器泛化铺路，几乎零额外成本），而不必等到全部 P1 阶段才做。

（AIdex 论文里晶系细分为 extinction group 分层分类的思路，也可以作为未来"晶系粒度是否要细化到空间群"讨论的参考，暂不影响当前 baseline。）
