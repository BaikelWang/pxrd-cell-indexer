## 0. 一句话

> **不是 FOM/架构小改能解决的：晶胞几何回归太粗，候选池在严口径下不够，评测/选模尺子偏松，且 `find_mapping` 会把子胞伪命中算进去。**

## 1. P0 — 必须先面对（否则不可能完胜引擎）

### P0-1 几何回归精度远低于硬指标

| 量 | 当前（量级） | 目标 |
|---|---|---|
| `angle_mae` / max-angle | ~10° / median **18°** | ≤ **3°** |
| `length_mape` / max-len-rel | ~16% / median **32%** | ≤ **5%** |
| 严口径 raw（MP100 / valid） | **3% / 6%** | 需支撑最终 Top-1 >68% |

角和边**同时**不够；`length_angle` 等 loss 消融未解决（valid 宽松最多 +1.5～+5.7pp，严口径仍崩）。

### P0-2 优化/选模尺子与北极星不对齐

- 训练 best ckpt 盯 **raw @ 0.3/10°**。
- 宽松下 valid fom ~86%、MP100 fom ~88% 曾造成「已超引擎」错觉。
- 同模型严口径：valid fom **8.6%**、MP100 fom **13%**。

**问题**：一直在优化「宽松命中」，不是「0.05/3° 命中」。

### P0-3 严口径候选池召回不足（真池，不是伪命中）

| 池 | 严口径 find_mapping topk | 逐参数真命中（约） |
|---|---:|---:|
| 默认 Top-20 | 26%（MP100）/ 16.5%（valid） | ~7% 量级 |
| 仅 8 Bravais snap | 12% | — |

引擎要 66%+，真池召回只有十几到二十几个点 → **理论天花板低于引擎**。

### P0-4 评测口径陷阱：`find_mapping` 子胞/倍胞伪命中

加密尺度后 find_mapping topk 可「涨」到 95%，但其中逐参数进 0.05/3° 的仅 **4/95**；其余体积比中位数 ~0.02。

**问题**：只报 find_mapping 会被假阳性误导；扩搜索前必须加**逐参数命中**或**体积比护栏**。

## 2. P1 — 结构性算法缺陷（已证实或强嫌疑）

### P1-1 低对称晶系系统性失败

- valid 宽松 proxy：cubic ~76%，hex/trig **3–7%**，mono/tric ~9–11%。
- hex/trig 角度常被拉向 ~90°。
- 拖垮整体均值的主要来源之一。

### P1-2 搜索空间过窄（相对经典 indexer）

- 仅 8 个 Bravais 假设；**无独立 mono/tri snap**。
- 尺度变体集合有限；盲目加密会被 P0-4 假阳性污染。
- 范式是「一次回归 + 小邻域」，不是 Mc/JADE 式大范围搜索。

### P1-3 Encoder 信息瓶颈（强嫌疑，待消融定量）

- 2θ 经 `long()` **取整到度**；最坏 |ΔQ| ~0.01–0.02 Å⁻¹ ≫ FOM `q_tol=1e-6`。
- 无波长输入；Transformer 很小（embed 32、2 层）。
- 尚未做「连续 2θ」训练消融，但是信息论上很硬的嫌疑。

### P1-4 训练目标不含 indexing 闭环

- Loss 只压六参数（或物理 MAE/Huber），**不监督**峰匹配 / Top-K / FOM / 晶格等价类。
- Huber δ=5° > 目标 atol 3°；无 5% 边长 hinge。
- Top-K/FOM 纯推理后处理，从不反传。

### P1-5 数据规模与难度不够

- 100k ≈ 全量 ~1.6%；10k→100k 宽松仅 +~4pp。
- 难晶系即使均衡抽样仍学不准。
- Phase3 最优宽松 raw ~**46.6%**（hlr0.001/bs128），仍远未触及严口径引擎线。

### P1-6 池变大后 FOM 仍弱（次主矛盾，但真实）

- 默认池严口径 fom 13%；密池（含伪命中）fom 仅 30%。
- 失败时错误大胞常有更高 `n_matched`；默认 heuristic 不用 de Wolff M / 强度。
- **在真池修好前不是第一刀**；修好后必须重抓。

## 3. P2 — 已知但非当前主因 / 已部分排除

| 问题 | 状态 |
|---|---|
| 峰(reduced) vs 标签(primitive) 设定错位（D1） | 探针显示**非主因**（raw 仍更贴 primitive）→ 暂缓 |
| 去晶系分类头 | Decision A：宽松 raw ~持平 → **排除为主因** |
| matrix9 vs matrix6 | ~0pp → **排除** |
| FOM 公式微调（宽松下） | 18 组 sweep 平台；严口径三 mode 同为 13% → **当前非主因** |
| 宽松 Top-K 召回 | ~99% → **宽松下排除**；严口径下仍是问题（见 P0-3） |
| IO 瓶颈 | 影响吞吐，不解释精度上限 |

## 4. 按层归类（算法 / 训练 / 数据 / 评测）

| 层 | 核心问题 |
|---|---|
| **算法** | 回归太粗；低对称角度偏置；搜索邻域小；Encoder 2θ 取整；FOM 在真难例上偏弱 |
| **训练** | 选模/loss 对齐宽松口径；无峰匹配/等价类监督；小幅 loss 消融无效于严精度 |
| **数据** | 规模小；难晶系难；域差 valid≠MP100（宽松）但严口径两边都崩 |
| **评测** | 曾用 0.3/10° 自欺；`find_mapping` 子胞伪命中膨胀 topk |

## 6. 建议攻关顺序（汇总级）

> 完整分阶段实验手册见 [`20260709-完胜引擎攻关方案v1.md`](20260709-完胜引擎攻关方案v1.md)（段 0→1→2→3 + Gate + 实验 A1–A6 / B1–B4）。

1. **定尺子**：双报宽松漏斗 + 严口径；严口径附带逐参数/体积比；训练至少监控严口径。  
2. **抬真池**：把严口径「真命中」topk 从 ~10–25% 拉向 70%+（回归精度 + 带体积约束的搜索/refine）。  
3. **再攻排序**：真池够了再强化 FOM（de Wolff / 强度 / 体积先验）。  
4. **数据 scaling + 难晶系**：与 2、3 并行。  
5. **Encoder 连续 2θ / 容量**：单独消融定量后再加码。
