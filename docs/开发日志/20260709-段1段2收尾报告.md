## 0. 一句话结论

在 100k + matrix6 基线上，**推理侧扩池/refine/Bravais、训练侧选模/loss/augment/过采样** 全部做完后：严口径真命中仍卡在 **valid raw elementwise ~3–7%、MP100 fom ~9–14%**，距引擎 **~55pp**。段1/段2 证明瓶颈是 **raw 晶胞几何精度**（`angle_mae≈9.2–9.8°`，远超 3°），不是 FOM、不是候选池宽度、也不是单点 loss/采样技巧。

**下一段主线：提高模型 raw 回归精度**（容量 / 数据规模 / 监督与表示），不再在推理后处理或轻量训练策略上赌大跃迁。

## 3. 段1 — 算法 / 推理侧（A1–A5）

**目标**：不重训或轻改编码，把严口径真池 / fom 抬过 Gate-1。  
**结果**：**Gate-1 失败**。

| 实验 | 做法 | 严口径关键结果 | 判决 |
|---|---|---|---|
| **A1** | elementwise / volume / gap 护栏 | 尺子落地；暴露伪命中 | ✅ 基础设施 |
| **A2** | 扩尺度集 + 池体积过滤；修 early truncate | topk_elem **卡死 7–8%**；无护栏 extended：map 94% / elem 7% / gap **87%** | ❌ |
| **A5** | soft Q-match L-BFGS refine | fom 最多 **+1pp**；池仍 7–8% | ❌ |
| **A3** | continuous 2θ pos + warm_start 10ep | MP100 topk_elem **8%→0%**；angle_mae 变差 | ❌ |
| **A4** | Bravais extended（mono + hex_strict） | topk_elem 持平/略降 | ❌ |

### 段1 机制结论

1. **扩池不能发明精度**：seed 离真解太远时，更大搜索只增加 mapping 伪命中。  
2. **局部 refine 无效**：median 角误差 ~10°+、边长相对误差远超 ±25–40% 合理盆地。  
3. **轻改 encoder 位置编码** 在短 warm_start 下破坏已学表示。  
4. **低对称 Bravais 假设扩展** 不解决 raw 几何粗的问题。

→ **推理侧上限已摸清**；必须动训练与表示能力。

## 4. 段2 — 训练策略（B1–B4）

**目标**：严口径选模 + loss/采样/增强，Gate-2（fom≥30% 且较段1最优 ≥+10pp）。  
**结果**：**Gate-2 失败**；仅 B4/B3 有小幅正向。

统一设定（除注明外）：100k matrix6，warm_start 链上 B4←baseline，8 epoch，`best_metric=strict_raw_top1_elementwise_rate`。

## 5. 跨段证据链：为什么「后处理 / 轻策略」不够

```
几何误差量级          严口径命中量级           引擎
─────────────────    ─────────────────      ────────
angle_mae ~ 9–10°  →  raw elem ~ 3–7%   →   需要 ≤3°
length_mape ~ 16%  →  topk_elem ~ 7–11% →   需要 ≤5%
                   →  fom ~ 9–14%       →   68% / 66%
```

| 杠杆 | 本轮最大 Δ（严口径） | 是否触及根因 |
|---|---|---|
| 扩池 / refine / Bravais | topk_elem ≈ 0；fom ≤+1pp | 否 |
| 严口径选模（B4） | valid elem **+2pp** | 弱对齐 |
| 关 2θ shift（B3） | valid elem **+1pp**；MP100 fom **+5pp** | 弱 |
| hinge / angle_heavy | 负向或崩塌 | 否（当前形式） |
| 低对称过采样 | −0.8pp | 否 |

**漏斗诊断不变**：宽松下 topk≈99%、fom≈88% → 排序已到头；严口径下池与 raw 同时崩 → **回归几何是唯一主瓶颈**。

## 7. Gate 判定

| Gate | 标准 | 实测最优 | 判定 |
|---|---|---|---|
| **Gate-1**（段1） | 严口径 `topk_elementwise` ≥ **40%** | ~**8–11%** | ❌ |
| **Gate-2**（段2） | 严口径 fom ≥ **30%** 且较段1最优 ≥+10pp | MP100 fom **14%**（B3a） | ❌ |
| 全量训练门禁 | Gate-1/2 过关后再 500k/6M | — | **仍禁止跳层全量赌** |

方案原「先算法再训练再 scaling」顺序正确；两段 Gate 都未过，说明 **必须升级「算法」定义：从推理后处理 → raw 回归能力本身**（容量、数据、监督），而不是继续在 Top-K/FOM/轻 loss 上打转。

## 8. 下一段：提高 raw 精度（建议议程）

> 用户确认方向：**目光聚焦到提高模型 raw 的精度**。

### 8.1 成功标准（建议）

| 阶段 | valid strict raw **elem** | `angle_mae` | 备注 |
|---|---:|---:|---|
| R0 诊断 | 建立误差分解（按 CS / 峰数 / 体积） | 分位数 | 不改模型 |
| R1 小步 | ≥ **15%** | ≤ **6°** | 证明可动 |
| R2 可用 | ≥ **35%** | ≤ **4°** | 再谈严口径 fom 冲 30%+ |
| R3 冲引擎 | raw/fom 路径清晰到 60%+ | ≤ **3°** | 才开全量生产线 |

### 8.2 优先假设（按杠杆排序，待开实验单）

1. **数据规模**：100k → 500k / 全量，在 **B3a 默认（无 shift + 严选模）** 上 scaling；回答「是不是纯数据不够」。  
2. **容量 / 表征**：更深/更宽 head；encoder 解冻策略与更长训；**慎重启** continuous 2θ（需从头或更长对齐，禁止短 warm_start 重蹈 A3）。  
3. **监督信号**：物理 loss **与** matrix6 SmoothL1 **相加**（吸取 B1b 教训）；可选 peak-Q 可微辅助（B1c，小 batch）。  
4. **低对称专项**：分阶段课程（B2c）或 per-CS 指标早停（B2b），而非单纯 ×2 行复制。  
5. **表示**：保持 matrix6；不优先 matrix9（Decision B 已否）。

### 8.3 明确不做（本阶段）

- 再扩 Top-K / refine 赌 Gate-1  
- 替换式纯物理 loss（B1b 形态）  
- 未过 R1 就启动 6.08M 长训生产线  
- 用宽松 0.3/10° 宣告「超过引擎」
