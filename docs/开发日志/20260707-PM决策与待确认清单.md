### D14. Top-K 与峰数处理：**参考 RealPXRD，但改成 indexing 任务口径**

| 项 | 结论 |
|---|---|
| **RealPXRD Top-K 来源** | 通过 `num_evals` 对同一输入多次 `model.sample()`，形成 best-of-N / Top-N 候选；常见 `num_evals=20` |
| **本任务建议** | baseline 可设计 Top-K 输出；训练时可先用 Top-K 平均 MSE/MAE 或 min-of-K 损失做候选方案，评测用 lattice match 判断是否命中 truth |
| **lattice match** | 参考 `起点.md`：三引擎 lattice 主指标 `ltol=0.3, atol=10°`；pymatgen `StructureMatcher` / `find_mapping` 可用于判断几何匹配 |
| **RealPXRD 峰数处理** | 仅做 `pxrd_y > 5` 过滤；**没有显式按峰数截断** |
| **重要修正** | `BertModel.max_seq_len=180` 主要约束 `pxrd_x.long()` 的 2θ embedding 索引范围（训练 2θ≤80），**不是 180 个峰的硬上限**；峰数过多主要带来 attention O(N²) 显存/速度问题 |
| **确认/记录时间** | 2026-07-07 |

### D9. 损失函数：**晶系条件化 mask 回归 + uncertainty weighting**

| 项 | 结论 |
|---|---|
| **决策** | 采纳文献调研建议（`20260707-文献调研-损失函数TopK冻结策略建议.md`） |
| **回归 mask** | Head 输出 6 个数 `(a,b,c,α,β,γ)`，但 **loss 只在该晶系的自由参数位置计算**（立方仅 `a`；正交 `a,b,c`；单斜 `+β`；三斜全部 6 个等），其余角度/长度确定性填充，不计入 loss |
| **回归 loss 形式** | 长度 `a,b,c`：MAPE 或 log 空间 MAE；角度：Huber/Smooth-L1 |
| **多任务配权** | 晶系 CE + lattice 回归用 **uncertainty weighting**（`Kendall et al. 2018` 可学习 σ 自动配权）；初版先固定 1:1 跑通，再切换 |
| **依据** | `Chitturi et al. 2021`（晶系条件化回归的 inductive bias）；`Kendall et al. CVPR 2018`（uncertainty weighting） |
| **确认时间** | 2026-07-07 |

### D15. Top-K 实现方式：**方案 A，7 个晶系头天然 Top-K**

| 项 | 结论 |
|---|---|
| **决策** | 共享 encoder；晶系分类头（7 类 softmax）+ **每个晶系一个独立回归子头**（只输出该晶系自由参数，见 D9） |
| **推理** | 按晶系分类概率排序，取 Top-K 个"晶系＋对应回归头输出"组合作为候选表 |
| **好处** | 候选天然跨晶系多样（对齐 McMaille/JADE 候选表形态）；训练稳定，无 WTA head collapse 风险；与已定 lattice match Top-K 评测（命中一个即算对）天然对齐 |
| **后续增强（不阻塞）** | 若晶系内部仍有倍胞/子胞模糊性，可后续叠加小型 **Annealed WTA** 子头（不用原始 WTA，避免头坍缩） |
| **依据** | `Guzman-Rivera 2012`／`Stochastic MCL 2016`／`Annealed MCL NeurIPS 2024`；`Chitturi 2021` 晶系条件化回归 |
| **确认时间** | 2026-07-07 |
