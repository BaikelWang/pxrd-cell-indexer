# 2026-07-07 — Agent 代码闭环纪律

## 背景

PM 强调：写完代码之后，Agent 必须 review，并且必须跑测试。

## 已写入 AGENT.md

新增 **代码闭环** 原则：

1. **Review**：写完代码后先自查 diff、接口、边界条件、是否误改无关文件。
2. **Test**：优先跑 `make test`；若只改局部，可跑最小相关测试，但最终交付前要说明测试范围。
3. **记录**：把验证命令、结果、已知风险写入开发日志或实验记录。

如果测试因环境、依赖、资源限制无法运行，必须明确说明原因、替代验证方式和剩余风险。

## 执行口径

- 测试未通过，不能声称完成。
- 未跑测试，不能模糊说“应该可以”。
- 代码改动完成后，最终回复必须包含验证结果或未验证原因。
