从 0 开始做一个深度学习任务，不要一上来就堆模型。最重要的是先把问题、数据、指标、baseline、训练流程、误差分析跑通。可以按下面这条主线理解：

**任务定义 → 数据处理 → baseline → 模型架构 → 训练策略 → 验证评估 → 消融分析 → 部署/复现**

---

## 1. 先把任务定义清楚

第一步不是写模型，而是回答几个问题：

这个任务到底是：

| 类型    | 例子            | 输出      |
| ----- | ------------- | ------- |
| 回归    | 预测材料带隙、磁矩、能量  | 连续值     |
| 分类    | 金属/半导体、是否稳定   | 类别      |
| 分割    | 图像语义分割、原子区域识别 | 像素/节点标签 |
| 生成    | 生成结构、分子、文本    | 新样本     |
| 排序/推荐 | 候选材料筛选        | 排名      |

然后明确输入输出：

例如材料性质预测任务：

[
\text{crystal structure} \rightarrow \text{band gap / magnetization / formation energy}
]

图像任务：

[
\text{image} \rightarrow \text{class label}
]

文本任务：

[
\text{text sequence} \rightarrow \text{label / embedding / response}
]

任务定义不清楚，后面模型再复杂也没用。

---

## 2. 数据是核心，不是模型

深度学习任务里，**数据质量通常比模型结构更重要**。

需要重点检查：

### 2.1 数据来源是否可靠

比如材料任务中，标签可能来自 DFT、实验数据库或文献。不同来源的计算参数、泛函、U 值、收敛标准不同，会造成 label noise。

### 2.2 数据是否泄漏

这是最常见错误之一。

比如：

* 训练集和测试集里有相同材料的不同构型；
* 图像增强后，同一张图的变体同时进入 train/test；
* 时间序列任务里，未来信息泄漏到过去；
* 文本任务里，答案模板泄漏到输入。

数据泄漏会导致测试结果虚高。

### 2.3 数据划分要合理

常见划分：

| 划分方式              | 适用场景                   |
| ----------------- | ---------------------- |
| random split      | 普通任务，数据独立同分布           |
| scaffold split    | 分子任务                   |
| composition split | 材料任务                   |
| time split        | 时间序列、金融、日志             |
| group split       | 同一来源/同一对象不能同时出现在训练和测试中 |

对于科研任务，不能只用 random split，因为 random split 往往太乐观。

### 2.4 标签分布要看

训练前一定要画：

* label histogram；
* 类别比例；
* 缺失值；
* 异常值；
* 输入尺寸分布；
* train/val/test 分布是否一致。

比如回归任务中，如果大部分样本 band gap 接近 0，模型可能只学会预测小值。

---

## 3. 一定先做 baseline

不要直接上 Transformer、大模型、复杂 GNN。

baseline 可以包括：

| 任务  | baseline                               |
| --- | -------------------------------------- |
| 回归  | 平均值、线性回归、Random Forest、MLP             |
| 分类  | majority class、Logistic Regression、SVM |
| 图像  | ResNet18                               |
| 文本  | TF-IDF + LR，BERT-base                  |
| 图任务 | GCN、GAT、GIN、CGCNN                      |

baseline 的意义是判断：

1. 数据有没有信号；
2. 复杂模型是否真的有提升；
3. 后续消融实验有没有参照。

一个很重要的原则：

**复杂模型必须打败简单 baseline，否则复杂性没有意义。**

---

## 4. 模型架构设计要围绕“归纳偏置”

架构不是越复杂越好，而是要匹配数据结构。

### 4.1 图像任务

图像有局部性和平移不变性，所以 CNN 很有效。

常见架构：

* CNN；
* ResNet；
* U-Net；
* Vision Transformer。

### 4.2 文本任务

文本是序列，有上下文关系，所以 Transformer 适合。

核心结构：

* token embedding；
* positional encoding；
* self-attention；
* feed-forward network；
* residual connection；
* layer normalization。

### 4.3 图结构任务

图数据没有固定顺序，有节点、边、邻接关系，所以要用 GNN。

常见架构：

* GCN；
* GraphSAGE；
* GAT；
* GIN；
* MPNN；
* Graph Transformer。

对于晶体材料任务，还要考虑：

* 周期性；
* 原子种类；
* 键长；
* 键角；
* 局域配位；
* 长程结构；
* 对称性；
* 多尺度结构。

所以材料图网络常见设计是：

[
\text{atoms} + \text{bonds} + \text{periodicity} \rightarrow \text{graph representation} \rightarrow \text{property}
]

### 4.4 架构设计不要只问“用什么模型”，而要问“为什么这个模型适合这个问题”

例如 SACGT 这类结构感知晶体图 Transformer，合理性在于：

* 晶体性质不仅由局域配位决定；
* 也受周期性、长程连接和局域畸变影响；
* 因此需要同时编码全局晶格结构和局部 motif 信息；
* 再用 attention 融合不同尺度特征。

这就是架构背后的物理/数据归纳偏置。

---

## 5. 模型细节要注意这些

### 5.1 输入特征

输入特征不能随便给。

例如原子特征可以包括：

* 原子序数；
* 电负性；
* 原子半径；
* 价电子数；
* 周期表族/周期；
* one-hot embedding；
* learned embedding。

边特征可以包括：

* 距离；
* cutoff 半径；
* 高斯展开；
* 键角；
* 周期镜像信息。

如果输入特征物理意义不清楚，模型可能学到伪相关。

### 5.2 归一化

不同特征尺度差异很大时必须归一化。

常见方法：

[
x' = \frac{x-\mu}{\sigma}
]

或者：

[
x' = \frac{x-x_{\min}}{x_{\max}-x_{\min}}
]

回归标签也常常需要标准化，训练结束再反归一化。

### 5.3 激活函数

常用：

* ReLU；
* LeakyReLU；
* GELU；
* SiLU。

Transformer/现代网络里 GELU 和 SiLU 很常见。

### 5.4 残差连接

深层网络容易梯度消失，残差连接很重要：

[
h^{(l+1)} = h^{(l)} + F(h^{(l)})
]

没有 residual，深层模型经常不好训。

### 5.5 Normalization

常见：

* BatchNorm；
* LayerNorm；
* GraphNorm。

CNN 常用 BatchNorm，Transformer 常用 LayerNorm，GNN 里 GraphNorm/LayerNorm 都常见。

### 5.6 Dropout

Dropout 用于防止过拟合，但不是越大越好。

小数据集上可以用 0.1–0.5，大模型常用 0.1 左右。

### 5.7 Pooling

图级任务必须把节点表示变成图表示。

常见：

[
h_G = \frac{1}{N}\sum_i h_i
]

也就是 mean pooling。

还有：

* sum pooling；
* max pooling；
* attention pooling；
* set2set pooling。

对于材料性质预测，mean pooling 和 attention pooling 都常用。注意：sum pooling 会强烈依赖原子数，适合总能量；mean pooling 更适合强度性质或归一化性质。

---

## 6. 损失函数要和任务匹配

### 6.1 回归任务

常用 MAE、MSE、Huber loss。

[
\mathcal{L}_{\text{MSE}} = \frac{1}{N}\sum_i (y_i-\hat{y}_i)^2
]

[
\mathcal{L}_{\text{MAE}} = \frac{1}{N}\sum_i |y_i-\hat{y}_i|
]

MSE 对异常值敏感，MAE 更稳健。

### 6.2 分类任务

二分类常用 Binary Cross Entropy：

[
\mathcal{L}_{\text{BCE}} =
-\frac{1}{N}\sum_i
\left[
y_i \log \hat{p}_i + (1-y_i)\log(1-\hat{p}_i)
\right]
]

多分类常用 Cross Entropy：

[
\mathcal{L}_{\text{CE}} =
-\sum_c y_c \log \hat{p}_c
]

注意：

* 二分类输出可以是 1 个 logit；
* 多分类输出是 num_classes 个 logits；
* 不要把 sigmoid 和 BCEWithLogitsLoss 重复使用；
* 不要把 softmax 和 CrossEntropyLoss 重复使用。

这是新手常犯错误。

---

## 7. 训练流程要规范

一个标准训练流程是：

```text
1. 固定随机种子
2. 加载数据
3. 划分 train/val/test
4. 数据预处理和归一化
5. 构建 DataLoader
6. 定义模型
7. 定义 loss
8. 定义 optimizer
9. 定义 learning rate scheduler
10. 训练
11. 验证
12. 保存 best checkpoint
13. 测试
14. 误差分析
15. 消融实验
```

训练时不是看 train loss，而是重点看 validation performance。

---

## 8. 优化器和学习率非常关键

常用优化器：

| 优化器     | 特点             |
| ------- | -------------- |
| SGD     | 稳定，但调参难        |
| Adam    | 默认好用           |
| AdamW   | Transformer 常用 |
| RMSprop | RNN/部分任务常用     |

现在大多数任务可以先用 AdamW。

学习率比模型结构还重要。

常见初始值：

| 模型          | 学习率         |
| ----------- | ----------- |
| MLP/CNN/GNN | 1e-3        |
| Transformer | 1e-4 到 5e-4 |
| 微调大模型       | 1e-5 到 5e-5 |

学习率太大，loss 震荡或 NaN；学习率太小，训练很慢或卡住。

常用 scheduler：

* StepLR；
* CosineAnnealingLR；
* ReduceLROnPlateau；
* Warmup + Cosine decay。

Transformer 一般建议加 warmup。

---

## 9. 训练过程中重点看这些现象

### 9.1 欠拟合

表现：

* train loss 高；
* val loss 也高；
* 模型学不动。

可能原因：

* 模型太小；
* 特征不够；
* 学习率太低；
* 训练轮数不够；
* 数据标签噪声太大。

解决：

* 增大模型；
* 加强特征；
* 调大学习率；
* 训练更久；
* 换更合适的架构。

### 9.2 过拟合

表现：

* train loss 很低；
* val loss 很高。

解决：

* 增加数据；
* 数据增强；
* dropout；
* weight decay；
* early stopping；
* 减小模型；
* 更合理的数据划分。

### 9.3 训练不稳定

表现：

* loss 变 NaN；
* 梯度爆炸；
* 指标大幅震荡。

解决：

* 降低 learning rate；
* 加 gradient clipping；
* 检查输入是否有 NaN；
* 检查标签是否异常；
* 检查 loss 写法；
* 检查归一化；
* 使用 mixed precision 时检查数值范围。

---

## 10. 指标不能乱选

### 回归指标

常用：

[
\text{MAE} = \frac{1}{N}\sum_i |y_i-\hat{y}_i|
]

[
\text{RMSE} =
\sqrt{
\frac{1}{N}\sum_i (y_i-\hat{y}_i)^2
}
]

[
R^2 =
1-
\frac{\sum_i (y_i-\hat{y}_i)^2}
{\sum_i (y_i-\bar{y})^2}
]

MAE 更直观，RMSE 更惩罚大误差，R² 反映整体拟合程度。

### 分类指标

常用：

* Accuracy；
* Precision；
* Recall；
* F1；
* ROC-AUC；
* PR-AUC；
* confusion matrix。

类别不平衡时，不要只看 Accuracy。比如 95% 都是负样本，模型全预测负类也有 95% accuracy，但没有任何意义。

---

## 11. 误差分析比单纯调参更重要

训练完后要看：

1. 哪些样本错得最离谱；
2. 错误是否集中在某一类；
3. 是否和数据分布有关；
4. 是否和结构大小、元素类型、能量范围有关；
5. 是否存在明显异常标签；
6. 模型是否只学到了 shortcut。

例如材料性质预测里，可以分析：

* 对大晶胞误差是否更大；
* 对磁性材料误差是否更大；
* 对过渡金属体系误差是否更大；
* 对小 band gap 材料是否误差更大；
* 对某些元素组合是否泛化很差。

这一步能指导你改模型，而不是盲目调参。

---

## 12. 消融实验一定要做

如果你提出一个新模型，必须证明每个模块都有用。

例如一个结构感知 GNN/Transformer，可以做：

| 实验                  | 目的                        |
| ------------------- | ------------------------- |
| 去掉 global branch    | 验证全局周期结构是否有用              |
| 去掉 motif branch     | 验证局部 motif 是否有用           |
| 去掉 attention fusion | 验证融合模块是否有用                |
| 换成 mean pooling     | 验证 attention pooling 是否有用 |
| 换成普通 GCN            | 验证整体架构优势                  |
| 改变 k-hop 大小         | 验证局域尺度影响                  |
| 改变 cutoff           | 验证邻接构造影响                  |

论文里如果没有消融，模型创新很难站住。

---

## 13. 可解释性不要最后才想

如果是科研任务，尤其是材料、物理、生物、医学任务，可解释性很重要。

常见方法：

* attention weight；
* gradient attribution；
* integrated gradients；
* SHAP；
* saliency map；
* node importance；
* edge importance；
* motif contribution；
* counterfactual analysis。

但要注意：

**attention weight 不一定等于物理因果贡献。**

所以最好结合：

1. 模型解释；
2. 物理先验；
3. 可视化；
4. 外部验证，例如 DFT、实验或文献。

---

## 14. 复现性要从一开始做

必须记录：

* random seed；
* 数据版本；
* train/val/test split；
* 模型超参数；
* optimizer；
* learning rate；
* batch size；
* epoch；
* scheduler；
* best checkpoint；
* 代码版本；
* 环境版本。

建议每次实验保存：

```text
config.yaml
metrics.json
train_log.csv
best_model.pt
test_predictions.csv
```

不要只保存最终结果，否则后面无法复现。

## 15. 常见新手错误

最容易出问题的地方有这些：

1. **没有 baseline，直接上复杂模型。**
2. **train/test 数据泄漏。**
3. **只看 accuracy，不看 F1、AUC、confusion matrix。**
4. **二分类输出、loss、label shape 不匹配。**
5. **学习率乱设。**
6. **没有归一化输入和标签。**
7. **test set 被反复调参使用。**
8. **只报最好一次结果，不报平均和方差。**
9. **没有消融实验。**
10. **把 attention 可视化直接解释成物理机制。**
11. **没有检查异常样本和错误样本。**
12. **模型复杂度和数据规模不匹配。**
13. **没有固定随机种子。**
14. **没有保存最佳 checkpoint。**
15. **没有记录实验配置。**

## 16. 一个比较规范的从 0 到 1 流程

可以按这个顺序做：

```text
Step 1：明确任务
输入是什么？输出是什么？回归还是分类？

Step 2：检查数据
数据量、标签分布、缺失值、异常值、类别比例。

Step 3：确定划分
train/val/test，避免数据泄漏。

Step 4：做最简单 baseline
平均值、MLP、GCN、ResNet18、BERT-base 等。

Step 5：建立训练脚本
保证 loss 能下降，验证集能正常评估。

Step 6：加入目标模型
先做最小可运行版本，不要一次加入所有复杂模块。

Step 7：调参
学习率、batch size、hidden dim、层数、dropout、weight decay。

Step 8：误差分析
找出模型在哪些样本上失败。

Step 9：消融实验
证明每个模块的必要性。

Step 10：可解释性分析
解释模型为什么有效。

Step 11：多随机种子复现
至少 3 次，最好 5 次。

Step 12：整理结果
主表、消融表、误差图、可解释性图。
```

## 17. 对科研型深度学习任务的核心建议

如果是写论文或做科研项目，最终要讲清楚四件事：

**第一，为什么这个问题重要。**

比如晶体性质预测中，传统 DFT 成本高，所以需要机器学习加速筛选。

**第二，为什么现有方法不足。**

例如现有 GNN 可能只强调局域邻接，不足以同时捕捉周期性、局域畸变和多尺度 motif。

**第三，你的模型为什么合理。**

模型结构要对应问题本身的物理机制或数据结构，而不是简单堆模块。

**第四，你的提升来自哪里。**

通过 baseline、消融、可解释性和误差分析证明模型确实有效。

---

## 最重要的一句话

从 0 做深度学习任务，真正的重点不是“选哪个最强模型”，而是：

**先让数据、任务、baseline、训练流程和评估体系完全可靠，再在这个基础上设计有问题针对性的模型架构。**

模型创新应该服务于任务机制，而不是为了复杂而复杂。
