# 2026-07-10 周报（W28）

> 项目：PXRD Cell Indexer（`PRXD-Cell-indexing-model-0706`）  
> 周期：2026-07-06 ~ 2026-07-10  
> 本周主题：从“宽松口径看起来很强”切换到“严口径真实能力诊断”，并完成 R0/R0b 根因定位

---

## 1. 本周目标

1. 对齐产品北极星口径（MP100, `ltol=0.05` / `atol=3°`），避免宽松指标误判。
2. 评估是否已具备“完胜 JADE/Mc”条件，明确主矛盾在算法/训练/数据的哪一层。
3. 给出下一阶段可执行路径，避免继续在非关键方向消耗算力。

---

## 2. 本周完成（关键里程碑）

### 2.1 M4 攻关主线推进

- 完成段 1（A1~A5）与段 2（B1~B4）收口复盘，形成统一攻关框架。
- 严口径下确认：当前方法尚未过 Gate-1/Gate-2，不能进入“全量数据直接冲榜”阶段。

### 2.2 R0：Raw 精度根因诊断（结构性问题坐实）

- 产出 `scripts/diagnose_raw_errors.py` 与完整诊断报告（B3a + baseline，valid1400 + MP100）。
- 核心结论：
  - strict 失败样本中 **85%** 为“边长与角度同时失败（both）”；
  - 命中几乎全来自 cubic，非立方 strict elem 仅 **0.17%**（2/1200）；
  - 出现显著 **pull-to-90°**（约 **75%** 样本）。

### 2.3 R0b：Loss 下不去问题的机制诊断

- 用 7 组小实验排除了“数据量/容量/2θ取整/歧义”等单因素假设。
- 关键定位：当前峰-token encoder 对“峰位→晶格”编码能力不足，输出趋向均值解（对应角度拉向 90°）。

### 2.4 R1 输入优化（Stage0→G2）完成并裁决

- histogram 输入分支验证成立；peak-token 方向否定。
- 100k G2 最佳（`hist_imin5`）：
  - strict elem **9.79%**
  - angle **9.06°**
  - 非立方 elem **~1%**
  - pull90 **67.5%**
- 结论：输入优化带来改善但不足以单独过 Gate，停止“仅输入线”继续扩规模。

## 4. 本周决策与结论

1. **不再继续“仅靠 FOM/扩池/局部 refine”路线**：这些方向不足以解决 raw 主矛盾。
2. **不直接上 500k/全量训练**：在 Gate 未过前扩数据性价比低、风险高。
3. **主线转向“非输入”层面**：晶系条件化、目标表示与监督设计，优先解决非立方与角度退化。
4. **周内形成统一共识**：宽松口径仅作漏斗参考，严口径作为主门禁。

---

## 5. 风险与阻塞

### 高风险

- 非立方样本几乎全灭，导致总体提升被 cubic 掩盖。
- 输出几何被强烈拉向正交邻域（90°），即使 Top-K/FOM 上调也难转化为 strict 命中。

### 中风险

- 若继续以宽松指标选模，容易重复“看起来进步、严口径无增益”的循环。

---

## 6. 下周计划（建议）

1. 启动“晶系条件化 + 目标表示”新一轮方案（先 10k/100k 小步 gate，再决定是否扩规模）。
2. 将非立方指标设为硬门禁（至少含：non-cubic elem、per-CS angle、pull90）。
3. 固化周度看板：strict raw / strict fom / non-cubic 三主指标并行追踪。
