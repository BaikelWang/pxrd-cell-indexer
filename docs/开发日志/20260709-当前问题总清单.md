# 2026-07-09 — 当前问题总清单

> **北极星**：MP100 @ `ltol=0.05` / `atol=3°` 完胜 JADE9（**68.1%**）与 McMaille（**65.9%**）。  
> **当前 NN（baseline）同口径**：fom **~9–13%**，raw **~3–6%** → 差约 **55–60pp**。

## 四、算法范式与模块缺陷

### 4.1 Encoder / 表示

| # | 问题 | 证据 | 优先级 |
|---|---|---|---|
| **A1** | **2θ 取整为整数度**，丢失亚度精度 | `bert.py` `pxrd_x.long()`；ΔQ~0.01–0.02 vs FOM q_tol 1e-6 | **P1**（待训练消融定量） |
| **A2** | 无波长 λ 条件化 | λ 写死在数据/FOM | **P2** |
| **A3** | Encoder 容量小（32-d embed，2 层） | 架构现状；未充分消融 | **P2** |
| **A4** | 一次点估计，非搜索式 indexing | 与 JADE/Mc 范式本质不同 | **P0**（战略层） |
| **A5** | 输出无晶体学约束（正定/Niggli） | matrix 解码可非物理 | **P2** |

## 八、问题关系（因果链）

```text
完胜引擎失败（0.05/3° 差 55–60pp）
│
├─【尺子】G1/G2/G3 口径错位与假阳性
│
├─【根因】R1–R5 几何回归太粗（角~18°、边~32%）
│     ├─ C1–C3 低对称更差
│     ├─ L1–L4 训练目标不对齐
│     └─ A1/A4 Encoder 精度损失 + 点估计范式
│
├─【放大】S1–S4 搜索窄；加密又被 mapping 假命中骗
│
└─【次级】F1–F2 池差时 FOM 非主因；池真变大后会变主因
         D1–D2 数据规模与难例不足
```

---

## 九、建议攻关顺序（浓缩）

1. **先修尺子**：双报 0.05/3° + 逐参数/体积比；训练监控严口径。  
2. **再攻几何**：把角/边误差压向 3°/5%（算法+loss+数据）。  
3. **再扩真搜索**：有体积约束的候选扩展，使**逐参数** topk 上升。  
4. **最后抠 FOM**：仅当真池召回接近引擎后。  
5. **并行数据 scaling**（500k/全量 + 低对称过采样）。

---

## 十、一句话

> **当前不是「差一点点调参」，而是：在错误的宽松尺子下显得还行，在引擎硬指标下几何回归与真候选池都远不够；评测还存在子胞假命中陷阱。必须同时改尺子、压误差、扩真搜索，才谈得上完胜 JADE/Mc。**
