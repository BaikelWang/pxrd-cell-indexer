> **目的**：在 Phase 0–3 实验收尾之际，固化「模型算法现状 + 实证问题 + 代码级理论缺陷」，作为后续逐项测试排查的基线文档。

## 1. 当前算法现状（实现快照）

### 1.1 管线一句话

```
峰表 (2θ, I)
  → RealPXRD BertModel encoder（2θ 取整为位置编码 + 强度为 token）
  → 512-d CLS embedding（可选 L2 normalize）
  → MLP 回归头 → 归一化 lattice（默认 matrix6 → 解码为六参数）
  → Bravais 8 假设 snap + 倍胞/子胞变体 → Top-20 候选
  → （推理时）FOM 峰匹配重排序 → Top-1
```

**训练只监督「直接回归」**；Top-K / FOM **不参与 loss**，仅推理后处理。

### 1.2 模块对照表

| 模块 | 实现位置 | 当前默认行为 |
|---|---|---|
| Encoder | `model/encoder/bert.py` + `loader.py` | RealPXRD 预训练；`embed_dim=32`，2 层 Transformer；`output_dim=512`；可 unfreeze |
| 回归头 | `model/heads.py` | `Linear(512→256)→GELU→Dropout→Linear→6`；**无晶系分类头**（Decision A） |
| 表示 | `data/normalization.py` | **matrix6**（规范矩阵自由分量 z-score）→ 解码六参数 |
| Loss | `losses.py` | 默认 baseline = 归一化 SmoothL1；另有 length_angle / cs_mask / cs_reweight / combined |
| Bravais Top-K | `model/bravais.py` + `topk.py` | 8 假设（cubic P/F/I、tet P、orth P、hex_trig_P、trig R、identity）+ 尺度变体；K=20 |
| FOM | `model/fom.py` | 默认 **heuristic**：`(-n_matched, volume, mean_ΔQ)`；λ 固定 CuKα；默认**不用**强度、**不用** de Wolff M |
| 训练选模 | `training/trainer.py` | `best_metric = top1_lattice_match_rate`（**raw**，默认 **ltol=0.3 / atol=10°**） |
| 评测 | `eval.py` / `eval_mp100.py` | pymatgen `find_mapping`；已支持自定义 `--ltol` / `--atol-deg` |

### 1.3 与传统 indexing 的本质差异

| | 本模型 | JADE9 / McMaille |
|---|---|---|
| 核心机制 | **一次回归**猜晶胞 + 小规模后处理搜索 | 从峰几何 **枚举/搜索** 晶胞 |
| 搜索空间 | Bravais snap + 少数尺度因子 | 大范围晶胞参数搜索 |
| 打分 | 简化 Q 匹配 FOM（默认 heuristic） | 完整 FOM / 系统消光等 |
| 输入精度 | Encoder 内 2θ **取整到度** | 全精度峰位 |
| 波长 | 隐式（训练数据固定 CuKα） | 显式参数 |

## 2. 实证问题（数字说话）

### 2.1 硬指标：严格容差下远未达到引擎

MP100，ideal 峰：

| 方法 | 口径 | Top-1 |
|---|---|---:|
| **JADE9** | 0.05 / 3° | **68.1%** |
| **McMaille** | 0.05 / 3° | **65.9%** |
| NN baseline（fom） | 0.05 / 3° | **13%** |
| NN `length_angle`（fom） | 0.05 / 3° | **7%** |
| NN baseline（raw） | 0.05 / 3° | **3%** |
| NN `length_angle`（raw） | 0.05 / 3° | **0%** |

差距约 **55–60pp**。宽松口径（0.3/10°）下 fom ~88%「超过」引擎，**是口径错觉，不能作为完胜判据**。

### 2.2 宽松口径下的漏斗（说明「排序已到头、回归不够」）

valid1400 / test1400（0.3/10°，matrix6 基线）：

| 阶段 | 约值 | 含义 |
|---|---:|---|
| raw_top1 | ~41% | 直接回归主瓶颈 |
| fom_top1 | ~86–88% | 后处理捞回 ~45pp |
| topk | ~99% | 候选池（宽松下）几乎饱和 |

严格口径下 topk 掉到 **21–26%** → 池内「命中」大量是宽松等价，不是 5%/3° 级贴合。

### 2.3 几何误差量级 vs 目标

| 量 | 当前（valid 基线量级） | 完胜引擎所需量级 |
|---|---|---|
| `angle_mae` | ~**10°** | **≤ 3°** |
| `length_mape` | ~**16%** | **≤ 5%** |
| 严口径 raw | 0–3% | 需逼近 / 超过 66–68%（经完整管线） |

### 2.4 晶系极不均衡

valid1400 proxy（每系 n=200）：cubic ~76%，hex/trig **3–7%**，mono/tric ~9–11%。  
低对称角度常被拉向 ~90°——结构性失败模式，不是随机噪声。

### 2.5 Phase 0–3 消融结论（截至目前）

| 改动 | 对宽松 raw 的影响 | 对严口径的启示 |
|---|---|---|
| 去分类头 / matrix9 | ~0 | 架构不是主因 |
| matrix6 | +1.2pp | 小收益，已采用 |
| `length_angle` | +1.5pp（valid） | MP100 宽松 raw 反降；严口径仍崩 |
| `cs_mask` / `cs_reweight` / `combined` | 均低于 baseline | 当前 loss 变体未解决根因 |
| FOM v2 | 宽松产品 Top-1 到 ~88% | 严口径仍远低于引擎 |

**一句话**：后处理与小幅 loss/表示消融，**抬不起严格几何精度**。

## 3. 理论缺陷分析（按代码模块）

以下按「假设 → 缺陷 → 为何阻碍完胜引擎」组织，均对应现有实现。

### 3.1 Encoder：信息瓶颈与精度损失

**实现要点**（`bert.py` `batch_input`）：
- 位置编码：`pxrd_x.long()` → **2θ 截断为整数度**
- Token：峰强度；子度级峰位信息在进入 Transformer 前丢失
- 容量：`encoder_embed_dim=32`，仅 2 层；无波长输入

**理论假设**：整数度峰位 + 强度足以反演 primitive 晶胞。

**缺陷**：
1. **峰位量化**：严格容差下 d 间距对 2θ 极敏感；取整等价于人为加噪声，与 JADE/Mc 使用全精度峰位不对等。
2. **无 λ 条件化**：2θ↔d 依赖波长；模型把 λ 吃进权重，无法显式校准。
3. **容量偏小**：低对称、多解歧义需要更强表示；当前 embedding 可能是信息瓶颈。
4. **点估计范式**：网络输出单一晶胞，而 indexing 本质是**搜索 + 打分**问题。

### 3.2 监督目标：与「晶格等价命中」不对齐

**实现要点**（`losses.py`）：
- baseline：归一化空间 SmoothL1
- physical：长度相对误差 + 角度 Huber（δ 默认 **5°** > 目标 atol 3°）
- **不监督**：晶格等价类（Niggli / 基变换）、峰匹配似然、Bravais 类型、Top-K/FOM

**理论假设**：压低六参数误差 ≈ 提高 `find_mapping` 命中率。

**缺陷**：
1. 六参数空间有**规范冗余**（等价描述）；loss 钉死某一个 jsonl 设定，不等于最大化等价命中。
2. Huber δ=5°、无「5% 边长 hinge」→ 优化边界与 0.05/3° **不一致**。
3. 训练从未对「峰表 ↔ 候选」闭环求导 → 无法直接优化 indexing 目标。
4. Phase2 消融已实证：换 physical loss **几乎不抬严精度**。

### 3.3 表示与标签：峰–胞设定可能错位

**实现要点**：
- 标签：primitive 六参数（`p_lattice_matrix`）
- 峰模拟路径（MP100 / 历史数据口径）：conventional → reduced → XRDCalculator
- matrix6：规范矩阵自由分量；解码无正定/Niggli 投影

**缺陷**：
1. **特征来自一种设定、标签来自另一种**（D1 已接受的风险）→ 回归学的是有偏映射，严容差下更致命。
2. 输出无晶体学约束 → 可产生非物理矩阵/角度，靠后处理 snap「硬掰」。
3. 全局归一化统计混合七晶系 → 难晶系角度分布被淹没。

### 3.4 Bravais / Top-K：搜索空间远小于经典 indexer

**实现要点**（`bravais.py` / `topk.py`）：
- 仅 8 假设；**无独立 monoclinic / triclinic snap**（低对称主要靠 identity + jitter）
- 尺度变体：`{2, 0.5, √2, 1/√2, √3, 1/√3}` + 单轴 ×2/×0.5
- confidence = `1/(1+几何偏差分)`；池不足时用 jitter 填充

**理论假设**：真解落在「回归点附近的 Bravais snap + 少数倍胞」内。

**缺陷**：
1. 经典工具在**大范围参数空间**搜索；本模型搜索半径由回归误差决定——回归偏 10° 时，snap 救不回 3° 命中。
2. 低对称覆盖弱 → 与 per-CS 极差一致。
3. 严口径下 topk 21–26%：说明**候选生成假设已破**——真解经常不在「可 snap 邻域」内。
4. jitter 填充可能污染 FOM 排序。

### 3.5 FOM：简化打分 ≠ McMaille / JADE

**实现要点**（`fom.py`）：
- 默认 heuristic：**不用** de Wolff M(N)、**不用**强度
- 仅匹配前 `n_lines=20` 观测峰；贪心 Q 匹配；λ 写死 1.54184 Å
- 体积作 tie-break（小体积优先）

**缺陷**：
1. 与专用引擎的 FOM / 系统消光 / 更完整峰利用不对等。
2. 宽松下 FOM 能把 topk≈99% 捞到 ~88%；严口径下池子本身只剩 ~25% → **FOM 再强也无米下锅**。
3. 默认忽略强度 → 丢掉区分近邻胞的重要信息（`intensity_weighted` 模式存在但非默认）。

### 3.6 训练与评测闭环：选模尺子错了

**实现要点**（`trainer.py` + `eval.py`）：
- 验证与 best ckpt：raw 上 `find_mapping`，默认 **0.3 / 10°**
- 训练过程**不算** Top-K/FOM，**不算** 0.05/3°
- 增强：2θ ±0.1°、强度缩放等 —— 与严格几何命中存在张力

**缺陷**：
1. **优化的指标不是要赢的指标** → 模型可在宽松 valid 上好看，在严口径 MP100 上崩盘（已发生）。
2. 增强鼓励对峰位扰动不敏感，可能与「必须压到 3°/5%」冲突。
3. 无端到端 indexing 训练信号（MCL / 峰似然 / ranking loss 等均未用，尽管设计文档曾引用相关文献）。

### 3.7 数据：规模、难度、域差

| 问题 | 证据 |
|---|---|
| 规模 | 100k ≈ 全量 1.6%；10k→100k 仅 +~4pp（宽松） |
| 难晶系 | hex/trig 极差；均衡抽样仍不够「学准」 |
| 域差 | 同 ckpt：valid 宽松 raw ~41% vs MP100 ~58%；严口径一律崩 |
| 理想棒谱 | 训练/MP100 均为 ideal sticks；部署实测谱更难，但当前连 ideal 严口径都未过 |

## 4. 缺陷优先级总表（供后续排查排序）

| 优先级 | 缺陷主题 | 类型 | 为何必须先面对 |
|---|---|---|---|
| **P0** | 几何回归精度不够（角/边误差 ≫ 3°/5%） | 算法+训练目标 | 严口径 raw/topk 崩塌的直接原因 |
| **P0** | 评测/选模口径未对齐 0.05/3° | 训练策略 | 否则优化方向持续跑偏 |
| **P1** | 低对称晶系（尤其角度）系统性失败 | 算法+数据 | 拖垮整体均值 |
| **P1** | 搜索空间过窄（Bravais+尺度 ≪ 经典搜索） | 算法 | 严口径 topk 只有 ~25% |
| **P1** | 数据规模不足 | 数据 | 已证明 scaling 有效但未做满 |
| **P2** | Encoder 2θ 取整 / 无 λ / 容量小 | 算法 | 信息论瓶颈，需消融验证贡献 |
| **P2** | 峰设定 vs primitive 标签错位 | 数据/标定 | 可能造成不可消的偏差 |
| **P2** | FOM 简化（相对引擎） | 算法 | 在池质量提升后才成为主矛盾 |
| **P3** | 已排除项：宽松下 Top-K、FOM 微调、分类头、matrix9 | — | 勿再作为主攻 |

## 5. 后续「一点一点测试排查」建议框架

> Phase03 sweep 跑完后按此清单逐项做**可证伪实验**（每项只改一个因素）。

### 5.1 先定尺子（不改模型）

1. 全量评测双轨：宽松 0.3/10°（漏斗）+ **严格 0.05/3°**（比引擎）。
2. 训练/early-stop 至少**监控**严口径（valid 子集或 proxy）；选模是否切换需单独决策。
3. 固定报告：raw / fom / topk × 两档容差 + length_mae / angle_mae + per-CS。

### 5.2 算法排查顺序（建议）

| 序 | 实验 | 要回答的问题 |
|---|---|---|
| A | Encoder 输入改为连续 2θ（或更高分辨率 bin） | 取整是否吃掉严精度？ |
| B | 角度专项 loss / 分晶系 head 或条件化 | 能否压 angle_mae→3° 量级？ |
| C | 扩大候选搜索（mono/tri 假设、更密尺度、局部 refine） | 严口径 topk 能否从 ~25% 拉回？ |
| D | 峰匹配式监督或 ranking loss（轻量） | 训练目标能否对齐 indexing？ |
| E | FOM 默认改 strict_dewolff / intensity_weighted | 在池质量足够时能否再涨 Top-1？ |
| F | 模型容量（更深 head / 更大 encoder） | 是否 compute-bound 表示不足？ |

### 5.3 数据排查顺序（建议）

| 序 | 实验 | 要回答的问题 |
|---|---|---|
| G | 500k / 全量 scaling（严口径监控） | 规模杠杆还剩多少？ |
| H | 低对称过采样 / 难例挖掘 | hex/trig 能否单独抬起来？ |
| I | 峰–标签设定对齐（primitive 峰或 conventional 标签）消融 | D1 错位贡献多大？ |

### 5.4 判定标准（每个实验）

- **主**：MP100 @ 0.05/3° 的 fom_top1（及 raw / topk）相对当前 baseline（fom 13%）是否上升。
- **辅**：valid1400 同口径；angle_mae / length_mape；per-CS。
- **对照**：JADE 68.1% / Mc 65.9% —— 未接近前不宣称「超过引擎」。

## 6. 总结

### 现状

我们有一条**可工作的**「回归 + Bravais Top-K + FOM」管线：在**宽松**容差下 valid/test 产品 Top-1 ~86–88%、topk ~99%，且曾误以为已超 Mc/JADE。

### 问题

在**严格**容差（完胜引擎的真实尺子）下，模型 Top-1 仅 **7–13%**，引擎仍有 **66–68%**。根因是**晶胞几何不够准**（尤其低对称角度），不是 FOM 没调好。

### 理论缺陷（浓缩）

1. **范式**：一次回归 + 小搜索，对抗的是经典大搜索 indexing。  
2. **目标**：loss/选模对齐宽松六参数误差，不对齐 0.05/3° 等价命中。  
3. **信息**：2θ 取整、无 λ、小 encoder。  
4. **搜索**：Bravais 集合偏窄，严口径下候选召回崩。  
5. **数据**：规模小、难晶系难、峰–标签设定可能错位。  
6. **打分**：FOM 简化；在池质量修复前不是第一矛盾。

### 下一步

等 `phase03_experiments`（length_angle 超参 sweep）结束后，按 §5 从「定尺子 → 算法消融 → 数据 scaling」逐项排查，每步留下可复现数字，直到严口径逼近并超过 JADE/Mc。
