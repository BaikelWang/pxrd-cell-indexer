## 1. 本周目标

1. 收口 R5/R6 支线，避免再在 Top-K/FOM/多锚上空转。

3. 验证 Encoder / Head / 输出表示三条主杠杆，找到能稳定抬升 strict 的生产基线。

## 4. 本周决策与结论

1. **Encoder 主线切换**：Peak Transformer（T48-geom）取代 histogram MLP。
2. **Head 主线切换**：`cs_conditional`（7 独立头）取代 FiLM / shared；收益来自分头容量，不是 clf 准确率（三者 ~89–90%）。
3. **关闭无效方向**：MCL（无假设选择器）、peak⊕hist fusion、rel-attn（P0 不过）、盲目加峰数、严口径 FOM/多锚池。
4. **方法论**：3500 只淘汰“完全不收敛”；**架构决策以 100k 为准**。
5. **输出表示**：`gstar6` 已证明值得保留为下一默认候选（相对 matrix6 +1.9pp）。
6. **产品距离**：相对 JADE/Mc（MP100 严口径 ~66–68%）仍远，但周内 strict **15%→42%**，主矛盾从“encoder 编码失败”转为“低对称/多解/输出坐标与搜索”。

## 5. 风险与阻塞

### 高

- **triclinic / monoclinic** 仍接近失效；总体数字仍被高对称抬高。
- 距引擎级产品门槛（MP100 strict ~68%）缺口仍大，单点回归未必够，后续需搜索/排序模块。

### 中

- MCL 显示“池内有更好解却选不出”——假设选择器未解。
- 3500 噪声导致误判风险；小规模消融不得再当架构裁判。
