# B2 实验报告：固定 B1 pool 上的候选排序对比（2026-07-21）

> 承接 `docs/开发日志/20260715-CellIndexing-可执行优化方案v3.md` §12、[B1-S1 实验报告](20260721-B1-S1-valid1400高对称验证.md)。
> 目的：B1-S1 已证明独立 q-search 候选池 recall 显著优于现有 NN 邻域搜索，但 pool recall 只是 Top-1 的上限——本轮验证「排序」这一步能否把 pool recall 兑现成 ranked Top-1 提升，并按 v3 §12.4 Gate 判定是否需要训练 learned reranker。

---

## 1. 一句话结论

**B2 Gate 通过，且不需要 learned reranker**：在固定的 B1 q-search Top-20 候选池（cubic/tetragonal/hexagonal/trigonal，各 40 个 valid1400 真实样本）上，三种排序方法——(1) NN proximity（纯粹信任 NN 初值，忽略峰拟合）、(2) 现有 R6-C legacy FOM（de Wolff 式启发式）、(3) 本轮新写的确定性 score（`search/rank.py`，覆盖 v3 §12.1 全部特征）——里，**(2) legacy FOM 和 (3) 新 score 都稳定通过 Gate**（ranking efficiency 89.3% / 86.4%，均 ≥75%；ranked Top-1 57.5% / 55.6%，均 ≥25%），且 **legacy FOM 略优于新 score**。关键前提是要给 legacy FOM 传入 NN 的体积先验（`ref_volume`）——不传的话它会系统性偏向"更小的等价胞"，在 cubic 上把 Top-1 拉低 12.5pp。**结论：不需要新写排序逻辑，只需要把现有 R6-C FOM 接上 NN 体积先验，就能把 B1 的 pool recall 兑现成 ranked Top-1。**

---

## 2. 方法

- 候选池：固定为 B1-S1 同一套独立 q-search 结果（oracle CS 路由，pool_budget=20，与 Gate 的 K=20 对齐——"固定 B1 候选池"，不重新生成/不扩大）。
- 样本：cubic/tetragonal/hexagonal/trigonal 各 40 个 valid1400 真实样本（与 B1-S1 同一套）。
- 三种排序：
  1. **`nn_proximity`**（v3 §12.3 "NN confidence" 的类比）：完全不看峰拟合，只按候选倒易度规与 NN 单点预测的（尺度归一化）距离排序——本质是"完全信任 NN 初值挑候选"。
  2. **`legacy_fom`**：现有 `model/fom.rerank_candidates_by_fom`（R6-C，de Wolff 式启发式），`ref_volume=NN 预测体积`（体积先验，公平对比的关键——见 §4）。
  3. **`deterministic`**（新写，`search/rank.py`）：按 v3 §12.1 列的全部特征构造的确定性分数——matched count（主）→ 半胞/超胞标记 → 理论峰密度惩罚 + `|log(V/V_nn)|` + 倒易度规到 NN 提案的距离（三者加权求和作为次级 tie-break）→ mean/median/max `|Δq|`。
- 判定：ranked Top-1 是否 strict elementwise 命中（ltol=0.05/atol=3°），以及 `ranking efficiency = ranked_Top1 / TopK(20)_recall`。
- 脚本：`scripts/run_b2_ranking_valid1400.py`；排序模块：`src/pxrd_cell_indexing/search/rank.py`；原始结果：`results/beat_engine/b1_search/b2_ranking_valid1400.json`。

---

## 3. 结果

### 3.1 整体（TopK recall 与 B1-S1 一致，作为 Top-1 的上限）

| | TopK(20) recall | nn_proximity Top-1 | legacy_fom Top-1 | deterministic Top-1 |
|---|:--:|:--:|:--:|:--:|
| 整体（n=160） | 64.4% | 46.9%（eff 72.8%） | **57.5%（eff 89.3%）** | 55.6%（eff 86.4%） |
| non-cubic（n=120） | 54.2% | 30.8%（eff 56.9%） | **45.8%（eff 84.6%）** | 42.5%（eff 78.5%） |

### 3.2 按晶系

| 晶系 | TopK recall | nn_proximity | legacy_fom | deterministic |
|---|:--:|:--:|:--:|:--:|
| cubic | 95.0% | 95.0%（100%） | 92.5%（97.4%） | **95.0%（100%）** |
| tetragonal | 55.0% | 42.5%（77.3%） | 40.0%（72.7%） | 42.5%（77.3%） |
| hexagonal | 82.5% | 35.0%（42.4%） | **75.0%（90.9%）** | 62.5%（75.8%） |
| trigonal | 25.0% | 15.0%（60.0%） | 22.5%（90.0%） | 22.5%（90.0%） |

（括号内为 ranking efficiency）

### 3.3 v3 §12.4 Gate 判定

| Gate 条件 | 要求 | legacy_fom | deterministic | nn_proximity |
|---|---|:--:|:--:|:--:|
| ranking efficiency（整体） | ≥75% | **89.3% PASS** | **86.4% PASS** | 72.8% FAIL |
| ranking efficiency（non-cubic） | ≥75% | **84.6% PASS** | **78.5% PASS** | 56.9% FAIL |
| ranked Top-1（整体） | ≥25% | **57.5% PASS** | **55.6% PASS** | 46.9% PASS |
| ranked Top-1（non-cubic） | ≥25% | **45.8% PASS** | **42.5% PASS** | 30.8% PASS |

**legacy_fom 和 deterministic 均通过 Gate；legacy_fom 略优。** `nn_proximity`（纯信任 NN 初值）在 non-cubic 上效率不足 75%（尤其 hexagonal 只有 42.4%），说明"只看 NN 初值不看峰拟合"在 NN 本身不够准的晶系上不够用——峰拟合信息（无论走 legacy FOM 还是新 score）是必要的。

---

## 4. 关键坑：legacy FOM 必须配 NN 体积先验才公平

第一轮跑的时候忘了给 `legacy_fom` 传 `ref_volume`（NN 预测体积），此时它在 `heuristic` 模式下退化为"偏好更小体积"（`vol_term = stats.volume`，见 `model/fom.py` `_sort_key_for_mode`）——这对 `nn_proximity`/`deterministic`（两者都吃了 NN 体积先验）是不公平的对比。修复前 vs 修复后（cubic）：

| | 修复前（无 `ref_volume`） | 修复后（`ref_volume=NN 体积`） |
|---|:--:|:--:|
| legacy_fom Top-1 | 80.0% | **92.5%** |
| legacy_fom efficiency | 84.2% | **97.4%** |

`ref_volume` 是 `FomRerankConfig` 已有的字段（专门为"避免偏好半胞"设计），本轮只是第一次把它接到 q-search 候选池的排序上——之前 legacy FOM 只用在 NN-seeded 的 Bravais 候选池（那种池子里所有候选天然聚在 NN 体积附近，这个先验价值不明显；q-search 候选池体积跨度大，先验的价值才显现出来）。

---

## 5. 决策

- **不需要新写排序逻辑，也不需要 learned reranker**：现有 `model/fom.rerank_candidates_by_fom`（`ref_volume=NN 预测体积`）已经通过 Gate，且优于本轮新写的确定性 score。
- 本轮新写的 `search/rank.py`（覆盖 v3 §12.1 全部特征：matched fraction、median/mean/max `|Δq|`、理论峰密度惩罚、半胞/超胞标记、倒易度规到 NN 距离）保留作为**备选/可解释性工具**——它在 cubic/tetragonal/trigonal 上与 legacy FOM 基本打平，只在 hexagonal 上明显更差（62.5% vs 75.0%），大概率是 `gstar_dist_to_nn` 权重过重（`RankConfig.gstar_dist_weight=5.0`，未调参）在 NN 本身偏差较大的晶系上把错误候选拉到了正确候选前面；本轮未做权重调优（v3 §12.3 允许用 valid1400 调参，但既然 legacy FOM 已经过 Gate，没有必要为了让新 score 更好而继续投入）。
- **推荐把 B1（独立 q-search）+ B2（legacy FOM + NN 体积先验）接成一条完整候选生成+排序流水线**，作为 B3（可选迭代 refiner）或后续放量评估的基础。
