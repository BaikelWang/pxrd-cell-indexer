# 2026-07-08 — 回归精度提升 Phase 0–3（单卡优先）

> **总目标**：在 100k 单卡上提升模型**直接回归** primitive 六参数的精度（主 KPI = `raw_top1`）；评测同时报告 raw / FOM / Top-K，避免「只看后处理天花板」掩盖回归本身是否变好。
>
> **范围**：Phase 0–3。不含模型容量放大、多卡 DDP、500k/全量 scaling（留 Phase 4–6）。
>
> **成功标准**：valid1400 上 `raw_top1` 相对 baseline **+5pp 起步**，且 `fom_top1` 不下降。

## 1. Phase 0 — 评测与训练指标对齐

### 做什么

**不改模型、不改 loss、不重训。** 只改「怎么量」：

1. 评测脚本同时输出四档指标：`raw_top1` / rerank `top1` / `fom_top1` / `topk`
2. 训练验证日志把整体 `lattice_mae` **拆成** `length_mae` + `angle_mae`
3. 验证阶段显式记录 `raw_top1_lattice_match_rate`
4. `best_metric` 继续锚定 raw 回归的 `top1_lattice_match_rate`（选 best ckpt 的标准不变）

涉及文件：`scripts/eval_valid.py`、`scripts/eval_mp100.py`、`src/pxrd_cell_indexing/eval.py`、`trainer.py`。

### 原理

此前存在**指标混用**：

- 训练选 best 看的是「直接回归」是否匹配
- 对外报告常看 FOM 重排后的 Top-1（~86–88%）
- 整体 `lattice_mae` 把 Å 与 ° 混在一个数里，看不出是长度差还是角度差

若不先拆开，后续改 loss / 超参时无法判断「回归本身变好了，还是只是后处理更吃香」。

### 目的

建立可复现的**双轨评测口径**，让后续每个实验都能回答：

- 直接回归有没有变准？（`raw_top1`）
- 候选池+FOM 天花板有没有掉？（`fom_top1` / `topk`）
- 误差主要来自长度还是角度？（`length_mae` / `angle_mae`）

### 结果 vs baseline

Phase 0 **本身不产生新模型**。用新口径重测旧 baseline，得到上表数字，作为后续消融的对照锚点。

| 变更点 | 是否影响精度数字 | 说明 |
|---|---|---|
| 双指标评测 | 否（只改报告） | 同一 ckpt，多报几列 |
| length/angle MAE 拆分 | 否（只改日志） | 不是把训练 loss 从 MAE 换成别的 |
| raw_top1 写入 valid | 否 | 方便训练曲线直接盯主 KPI |

## 2. Phase 1 — 数据管线 profiling 与 DataLoader 优化

### 做什么

1. DataLoader 打开 `prefetch_factor` / `persistent_workers`
2. 新增 `scripts/profile_dataloader.py`：短跑对比 **data_wait vs compute**
3. Trainer 可选首 epoch 计时（`optim.profile_timing`）

### 原理

单卡训练若 **IO 等待 >> GPU 计算**，加卡/加深模型收益有限，应先修数据管线；若已 compute-bound，再考虑多卡 DDP。

### 目的

回答「现在该不该上 4 卡」：用实测瓶颈做决策门，而不是凭感觉。

### 结果 vs baseline

| 项 | 数值 |
|---|---|
| data_wait | **38.8 ms/step** |
| compute | **5.1 ms/step** |
| 瓶颈 | **IO**（约 7.6× compute） |

**结论**：单卡 compute 远未饱和 → **本轮不做 DDP**；后续若要加速，优先 LMDB/缓存/worker，而不是先堆 GPU。  
对精度：**无直接影响**（管线优化不改变 loss/模型，只影响吞吐）。

## 3. Phase 2 — 损失函数升级（逐项消融）

### 做什么

在**同一数据、同一模型结构、同一超参骨架**下，只改 `loss.mode`，各训一轮 100k（最多 20 epoch + early stop），训完用 dual-metric 评测。

| mode | 监督什么 | 原理一句话 |
|---|---|---|
| `baseline` | 归一化 matrix6 SmoothL1 | 历史默认；各维等权，在归一化空间回归 |
| `length_angle` | 物理空间：长度**相对误差** + 角度 **Huber** | 直接压 Å/% 与 °，避免归一化尺度掩盖物理误差 |
| `cs_mask` | 物理空间 + 晶系 mask | 对称固定的参数（如立方 a=b=c）不进 loss，避免「惩罚不该学的维」 |
| `cs_reweight` | 归一化 SmoothL1 + 难晶系样本加权 | hex/trig 等难例加权，缓解晶系不平衡 |
| `combined` | baseline 回归项 + 物理 mask/reweight | 归一化稳定性 + 物理约束的折中 |

配置：`configs/scale_100k_loss_{length_angle,cs_mask,cs_reweight,combined}.yaml`。

### 原理

直接回归卡住在 ~41% raw，可能原因包括：

1. **归一化空间与物理误差不对齐**（小晶胞相对误差被低估）→ 试 `length_angle`
2. **对称约束维被错误惩罚** → 试 `cs_mask`
3. **难晶系样本被淹没** → 试 `cs_reweight`
4. 以上组合 → 试 `combined`

主 KPI 仍是 **raw_top1**；`fom_top1` 只作「后处理天花板是否掉」的护栏。

### 目的

找出哪种 loss 能把 **直接回归** 抬上去（目标 +5pp），同时不牺牲 FOM 天花板。

## 4. Phase 3 — 超参 sweep

### 做什么

在 Phase 2 **胜出的 loss mode** 上，网格搜索 `head_lr × batch_size`（脚本：`scripts/sweep_train_hyperparams.py`），再在 valid1400 + test1400（及可选 MP100）做 dual eval。

### 原理

loss 形态定了之后，学习率与有效 batch 仍可能差几个百分点；小网格比盲目加大模型便宜。

### 目的

在已选 loss 上榨取剩余增益，确认是否逼近 / 超过 +5pp 目标。

### 结果 vs baseline

**尚未开始**（等 Phase 2 选出最优 loss 后启动）。

## 9. 精度影响因素分析（多角度，2026-07-09）

> 基于现有实验结果，回答：**当前究竟是什么在限制模型精度？**  
> 主锚点：`scale_100k_no_cs_matrix6_seed42`；主 KPI：`raw_top1`（valid1400 ≈ **40.9%**）。

### 9.1 管线漏斗：问题在哪一段？

| 阶段 | valid1400 | test1400 | 含义 |
|---|---:|---:|---|
| **直接回归 raw_top1** | **40.9%** | ~41.8% | **当前主瓶颈** |
| Bravais 置信度 Top-1（无 FOM） | ~40.3% | — | 与 raw 几乎同级 |
| FOM v2 Top-1 | **85.9%** | **87.6%** | 排序把 ~45pp 捞回来 |
| Top-K 召回 | **99.1%** | **99.8%** | **不是瓶颈** |

漏斗解读：

1. **真解几乎一定在 Top-20 里**（~99%）→ 候选生成 / Bravais 展开够用。
2. **FOM 已把产品 Top-1 推到 ~86–88% 天花板**；再抠 FOM 收益极小（18 组 sweep 几乎不动）。
3. **直接回归只有 ~41%** → 若目标是「模型本身更准」，必须攻 raw，而不是再调后处理。

入池后的历史排序病（FOM 前）：真解在池内时仅 **39.4%** 排到 rank-1，**91.4%** 挤在 rank 1/3/4——已被 FOM 基本治好；**剩下来的 ~12% 在池未登顶**，多与倍胞/近邻混淆有关，不是 raw 主矛盾。

### 9.2 架构与表示：已经试过、影响很小

| 决策 | raw/Top-1 Δ | 结论 |
|---|---:|---|
| 去掉晶系分类头（Decision A） | −0.6pp | **不是瓶颈** |
| angles → matrix6（Decision B） | **+1.2pp** | 有小收益，已采用 |
| matrix6 → matrix9 | ~0pp | **不是瓶颈** |

架构消融说明：当前 head/表示形态不是 41% 的主因；继续在「加分类头 / 换 9 维」上打转性价比低。

### 9.3 损失函数：能动一点，但远不够，且有域偏移风险

| loss | valid raw | Δ | valid fom | MP100 raw | MP100 fom |
|---|---:|---:|---:|---:|---:|
| baseline | 40.9% | — | 85.9% | **58%** | 88% |
| `length_angle` | 42.4% | **+1.5pp** | 87.9% | 47% | **92%** |
| `cs_mask`（训练中） | ~38% @ep20 | 暂低于基线 | — | — | — |

要点：

- 物理空间 loss 在 valid 上只换来 **+1.5pp raw**，未达 +5pp 目标。
- 同一改动在 MP100 上 **raw −11pp、fom +4pp** → 优化目标与「直接回归命中」不完全同向；可能让点更利于 FOM 排序，却更偏离严格几何命中。
- `length_angle` 的 valid `length_mae`（1.63Å）反而高于 baseline（1.03Å），但 `angle_mae` 略好（9.3° vs 9.8°）→ **角度略好、长度变差**，与 MP100 raw 下降一致。

### 9.4 晶系难度极不均衡（最硬的结构性问题之一）

valid1400 上 baseline 的 per-CS proxy / MAE（每系 n=200）：

| 晶系 | lattice_mae | proxy match | 相对难度 |
|---|---:|---:|---|
| cubic | 2.99 | **76.5%** | 易 |
| tetragonal | 4.59 | 24.5% | 中 |
| orthorhombic | 4.58 | 16.5% | 中 |
| monoclinic | 5.28 | 9.0% | 难 |
| triclinic | 6.69 | 11.0% | 难 |
| hexagonal | 5.66 | **3.5%** | **极难** |
| trigonal | **8.17** | **7.0%** | **极难** |

历史 10k 诊断也一致：hex/trig 的 α/γ 常被拉向 ~90°（真值常在 100°+），角度 MAE 可达 14–25°。  
100k scaling 时「晶系对、晶格错」仍占 rank-1 失败的 **37.5%**——低对称晶格几何本身没学好。

### 9.5 容差口径 vs 真实精度（数字好看，几何仍粗）

正式匹配：`ltol=0.3`（边长 30%）、`atol=10°`（与 Mc/JADE 对照一致）。

MP100 baseline 容差扫描（同一 ckpt）：

| 口径 | raw_top1 | fom_top1 |
|---|---:|---:|
| **0.3 / 10°（现行）** | **58%** | **88%** |
| 0.2 / 8° | 34% | 82% |
| 0.1 / 5° | 11% | 54% |
| 0.05 / 2° | 2% | 9% |

补充：

- valid：`length_mae≈1.03Å`，`length_mape≈16%`，`angle_mae≈9.8°` —— 平均误差已经顶在 10° 容差边缘。
- MP100 raw **命中样本**的平均角度误差仍有 **7.5°**，相对边长误差 **~20%**；**未命中**则角度均值 **14°**。
- raw 未命中中，**角度单独超限**（20/42）多于长度单独超限（4/42）→ **角度误差是更常见的杀伤项**。
- proxy match（21%）≪ pymatgen raw（41%）→ 约 20pp 靠 cell/subcell 映射「救」回来，说明不少命中是宽松几何等价，不是逐参数贴合。

**结论**：现行口径适合对外比引擎；若问「回归精不精」，必须看 MAE + 更严容差——那里模型还很粗。

### 9.6 数据规模与域偏移

| 设置 | 指标 | 数字 |
|---|---|---:|
| 10k → 100k | Top-1 lattice | 35.4% → 39.7%（**+4.3pp**） |
| 10k → 100k | joint Top-1 | 20.9% → 27.4%（**+6.5pp**） |
| 同一 100k matrix6 ckpt | valid raw vs MP100 raw | 40.9% vs **58%**（**+17pp 域差**） |
| 训练数据占比 | 100k / ~608 万 | ~1.6% |

含义：

1. **加数据有效**（D29 已判定），但 10× 数据只换 ~4pp lattice → 收益在衰减，仍远未饱和全量。
2. **MP100 比 valid1400 好做**（或分布更易），不能把 MP100 58% 当成 valid 已解决。
3. IO profiling 显示训练仍 IO-bound，但不解释精度上限。

### 9.7 什么已经可以排除？

| 因素 | 证据 | 判定 |
|---|---|---|
| Top-K 召回不足 | 99%+ | **排除** |
| FOM 公式未调优 | v2 后 18 组 sweep 平台 | **接近排除**（产品 Top-1） |
| 晶系分类头缺失 | Decision A −0.6pp | **排除** |
| matrix9 自由度不够 | vs matrix6 ~0 | **排除** |
| Bravais 约束错误 | 100k 约束验证通过 | **基本排除** |

### 9.8 综合排序：什么最影响当前精度？

| 优先级 | 因素 | 证据强度 | 主要影响指标 |
|---|---|---|---|
| **1** | **几何精度不够：角度/边长误差远大于 3° / 5%** | 极强 | 严口径 Top-1；angle_mae≈10°；length_mape≈16% |
| **2** | **低对称晶系（hex/trig/mono/tric）学不准** | 很强 | per-CS proxy 3–11% vs cubic 76% |
| **3** | **训练规模仍小（100k ≪ 全量）** | 很强 | 10k→100k 仅 +4pp；全量未用 |
| **4** | **优化目标与「完胜引擎」不对齐** | 强 | 训练盯宽松 raw；loss 消融未抬严口径 |
| **5** | **候选池在严口径下召回崩塌**（topk 100%→21–26%） | 强 | 严口径 topk；说明池内也是宽松命中 |
| **6** | 模型容量（head/encoder）未充分试 | 中 | Phase 4 |
| **7** | Top-K/FOM/分类头/matrix9（宽松口径下） | 已排除或很弱 | — |

### 9.9 北极星重标定：完胜 JADE9 / McMaille

**硬指标（MP100，ideal 峰）**：`ltol=0.05`，`atol=3°` 下 Top-1  
- 必须超过：**JADE9 68.1%**、**McMaille 65.9%**  
- 当前 NN：fom **7–13%**，raw **0–3%** → 差距约 **55–60pp**

**宽松口径（0.3/10°）不再作为「是否超过引擎」的判据**——它只适合看相对消融与漏斗。此前 fom ~88%「超引擎」是口径错觉。

### 9.10 问题分层（算法 / 训练 / 数据）

| 层 | 当前问题 | 证据 | 后续方向（研究级） |
|---|---|---|---|
| **算法** | 回归输出几何太粗；低对称角度系统性偏 90°；严口径下候选也不够贴 | MAE、per-CS、严口径 topk 崩 | 角度/晶系条件化建模；更贴峰匹配的监督或 refine；容量放大 |
| **训练策略** | loss 与严口径命中不对齐；100k + 小幅 loss 消融几乎不动严精度 | Phase2 最大 +1.5pp（宽松）；严口径仍个位数 | 以 0.05/3° + MAE 为训练/选模信号；分晶系/角度加权；更长程 schedule |
| **数据** | 只用 ~1.6% 全量；难晶系仍难；域差 valid≠MP100 | scaling +4pp；hex/trig 极差 | 500k→全量；难例/低对称过采样；峰增强贴近 deploy |
| **评测** | 曾用宽松口径自欺 | 严口径引擎 66–68% vs NN 7–13% | **双轨**：严口径比引擎；宽松口径看漏斗 |

### 9.11 一句话诊断

> **不是「排序不够」或「架构选错」，而是「晶胞几何回归精度不够」——尤其在低对称晶系的角度上；再叠加数据规模不足与优化目标偏宽松。**  
> 在 `0.05/3°` 下，专用引擎仍有 ~66–68%，我们只有个位数到十几个点；要完胜 JADE/Mc，必须把误差从「~10° / ~16% 边长」压到「≤3° / ≤5%」量级，并让严口径下的候选池召回从 ~25% 回到高位。
