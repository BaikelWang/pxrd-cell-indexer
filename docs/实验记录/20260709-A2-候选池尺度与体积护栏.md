## 1. 假设

默认 `{2,0.5,√2,√3}` 尺度不足；但无护栏的密尺度会产生 `find_mapping` 伪命中。带体积护栏的扩展尺度可抬升 **`topk_elementwise` @ 0.05/3°**。

**成功标准（方案）**：较 A2a baseline 逐参数 topk **绝对 ≥+10pp**，且伪命中率可控。

## 5. 结论

1. **假设部分否定**：扩展尺度 **不能** 抬升真命中。`topk_elementwise` 卡在 **7–8%**，相对 A2a **+0～1pp**，远低于 **+10pp** 门槛。  
2. **无护栏扩池 = 伪命中爆炸**：A2b `topk mapping` 94%，但 elementwise 仍 7%，gap **87%** —— 与 Phase3 密尺度诊断一致，现已用正式护栏指标复现。  
3. **体积护栏有效压伪命中**：A2c–e gap 从 87% 降到 **6–9%**，但真池几乎不涨 → 问题不在「缺尺度」，而在 **raw 几何离真解太远**，尺度变体够不到 0.05/3° 邻域。  
4. **FOM 在假池上会抬 mapping、不抬 elementwise**（A2b fom 35% vs elem 0%）；A2e 略见 fom_elem=3%，仍可忽略。  
5. **Gate-1 路径**：仅靠 A2 无法把 `topk_elementwise` 拉到 40%。必须进入 **A3（Encoder）/ A4（低对称）/ A5（refine）** 抬 raw 或局部搜索。

**决策**：A2 **关闭扩池作为主攻**；保留代码能力（`--scale-set` / pool volume filter）供后续与 A5 组合；默认推理仍可用 `default` + 无 pool 体积过滤（评测侧继续双报 elementwise）。

## 7. 附录：体积护栏语义

- **评测侧** `*_volume_guarded_rate`：相对 **truth** 体积（A1）  
- **池构建侧** `pool_max_log_volume_ratio`：相对 **raw 预测** 体积（A2）  
两者用途不同：前者防伪命中统计，后者防池内塞入离谱超胞。
