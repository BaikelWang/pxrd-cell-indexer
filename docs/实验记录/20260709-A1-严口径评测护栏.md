## 1. 假设

此前部分「突破」（尤其宽松 FOM Top-1、密尺度 topk）可能混入 `find_mapping` 子胞/超胞伪命中。统一输出：

| 尺子 | 含义 |
|---|---|
| `*_lattice_match_rate` | pymatgen `find_mapping`（历史主指标） |
| `*_elementwise_rate` | a,b,c,α,β,γ 逐参数在 ltol/atol 内（拒子胞） |
| `*_volume_guarded_rate` | `find_mapping` ∧ \|log(V_pred/V_truth)\| ≤ log(2) |
| `*_mapping_vs_elementwise_gap_rate` | mapping 命中但 elementwise 失败的比例（伪命中率） |

## 5. 结论

1. **假设成立**：宽松口径下 `find_mapping` 与 elementwise 差距巨大。  
   - MP100 topk：100% vs **44%**（gap **56pp**）  
   - valid topk：99.1% vs **37.4%**（gap **61.7pp**）  
2. **FOM 宽松「高分」几乎全是伪命中**：fom find_mapping ~86–89%，但 **fom elementwise ≈ 0–1%**。产品 Top-1 在宽松尺子下被严重高估。  
3. **严口径真池极低**：`topk_elementwise` 仅 **8%（MP100）/ 10.7%（valid）**，距 Gate-1（≥40%）差约 **30pp**。  
4. **严口径下 raw 伪命中较少**（gap 1–3%），主矛盾是几何不够准 + 池内无真解，而非 mapping  alone。  
5. **体积护栏**在宽松下把 topk 从 100%/99% 压到 83%/82%，仍远高于 elementwise → 2× 体积护栏不够严，A2 需与 elementwise 联用。

**成功标准（方案 §7.1）**：脚本可复现双轨 + 护栏；后续实验统一引用 `results/beat_engine/` → ✅。
