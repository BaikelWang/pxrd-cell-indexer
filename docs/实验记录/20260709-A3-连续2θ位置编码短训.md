# 实验 A3 — Encoder 连续 2θ 位置编码（短训）

> **日期**：2026-07-09  
> **状态**：✅ 完成（**未达成功标准；严口径变差**）  
> **总纲**：[`20260709-完胜引擎攻关方案v1.md`](../开发日志/20260709-完胜引擎攻关方案v1.md) §7 A3  
> **单因素**：`position_encoding: continuous`；其余 loss/Top-K/FOM 不变  
> **前置**：A1/A2/A5 推理侧均未抬严口径

---

## 1. 假设

`bert.py` 中 `pxrd_x.long()` + `nn.Embedding(180)` 把 2θ 截成整数度，丢失子度级峰位，是严精度瓶颈之一。改为连续 float 2θ → MLP 位置编码可改善几何回归。

**成功标准**：严口径逐参数 raw **≥+3pp**，或 `angle_mae` 降 **≥2°** 且边长 MAPE 不恶化 >2pp。

---

## 2. Code review 要点

| 发现 | 处理 |
|---|---|
| `embed_positions = Embedding(max_seq_len=180)`，2θ 当索引 | 新增 `position_encoding ∈ {discrete, continuous}` |
| continuous 与预训练 Embedding 形状不兼容 | warm_start 跳过 `embed_positions`；其余 41 个 key 加载 |
| A3-i（×100 取整）会把 Embedding 表扩到 ~9000 | **未做**；优先 continuous（更干净） |

**改动**：`bert.py`、`ModelConfig.position_encoding`、`warm_start_checkpoint`、`configs/scale_100k_a3_continuous_pos.yaml`

## 5. 结论

1. **假设否定（本轮设置）**：连续 2θ 短训 **未改善** 严口径；全面变差。  
2. **根因推断**：位置编码从 Embedding→MLP 是结构性断裂；warm_start 保留了「按整数度训练」的下游权重，与连续输入分布不匹配；10 epoch 不够让新 pos MLP + encoder 对齐。  
3. **angle_mae 未降反升**（+0.88°），不满足「降 ≥2°」旁路标准。  
4. **A3 降级为 P2**：不作为当前主攻；若再试需 **从头训 continuous（无 warm_start）或更长 schedule**，且选模改严口径（B4）。  
5. **段 1 现状**：A1✅尺子 / A2❌扩池 / A5❌refine / A3❌连续编码 → **推理侧与轻量编码改动均未打开 Gate-1**。
