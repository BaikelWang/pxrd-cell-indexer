# B1+B2 · MP100 基线复评（2026-07-21）

> 目的：判定本轮 B1（独立 q-search）+ B2（legacy FOM + NN 体积先验）是否足以更新生产 baseline。  
> 脚本：`scripts/run_b1b2_mp100.py`；产物：`results/beat_engine/b1_search/b1b2_mp100.json`。  
> 口径：niggli 标签 / strict elementwise `ltol=0.05` / `atol=3°` / n=100。

---

## 1. 一句话结论

**可以更新「搜索/推理侧」baseline，但不能替换「模型单点」baseline。**

- 锁定模型 baseline（A3-G1 raw，S0）：MP100 strict **23%**（口径修复复跑已对齐）
- B1+B2（NN 预测 CS，可部署）Top-1：**36%**（**+13pp**）
- B1+B2（oracle CS，上限）Top-1：**42%**（**+19pp**）
- 高对称子集（n=60）：NN raw 36.7% → B1+B2 pred **51.7%** / oracle **61.7%**
- 低对称（n=40）仍弱：merged **12.5%**（NN raw 2.5%）——顺序求解未就绪，不能宣称全量问题已解决

相对 JADE 68% / McMaille 66% 仍差 **~26–32pp**，北极星远未追上。

## 3. 能否更新 baseline？

| 层级 | 决策 | 新数字 |
|---|---|---|
| **模型单点（A3-G1 raw）** | **不改** | MP100 strict **23%**（S0 锁定，复跑对齐） |
| **高对称候选流水线** | **更新** | B1+B2 pred-CS Top-1 **36%**（全量）/ **52%**（高对称） |
| **全量生产 KPI 整表替换** | **暂不** | 低对称未就绪；CS 路由 −6pp；距 JADE 仍 >25pp |

建议文档写法（双轨）：

```text
模型 baseline：A3-G1 raw · valid1400 ~40.6% · MP100 23%
推理侧（高对称可用）：B1 q-search + B2 FOM(ref_volume)
  MP100 Top-1 36%（pred CS）/ 42%（oracle CS）
```

---

## 4. 含义

1. **S2 在北极星上第一次给出实质增益**（相对 A3-G1 +13pp 可部署 / +19pp 上限），比 S3 的 +0.7pp 有意义得多。  
2. 增益几乎全部来自**高对称**；低对称仍接近失效 → 下一刀仍是顺序求解 + trigonal 设定修复。  
3. CS 路由有 −6pp 头寸：部署时可用「NN CS 软先验 + 多系并行搜」而不是硬路由。  
4. 排序效率接近顶满 pool（oracle Top-1 42% vs pool merged 49%）：瓶颈仍是**候选覆盖**，不是 FOM。
