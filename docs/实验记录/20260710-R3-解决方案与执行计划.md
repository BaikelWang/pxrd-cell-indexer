# R3 解决方案与执行计划（可部署路径）

> 2026-07-10 · 相对 R2 cs_pred 13.57% 继续冲 Gate（elem≥15% / ang≤6° / 非立方≥5% / pull90≤55%）

## 一句话方案

**保留 histogram + CS 条件化；把 cubic 再拆成 3 个 Bravais 设定头；用 setting classifier 做可部署路由；用小权重 `joint_phys` 抬非立方；暂不做 500k/Top-K/FOM。**

## 为什么是这条路

1. **R2 已证明**：CS 分头有效（13.57%），但同一 cubic 头混训 90°/60°/109.47°，角被拉向均值。  
2. **R3 oracle 已证明**：cubic 分头 + GT setting → **15.93%**（过 elem Gate），cubic 94%，pull90 54.6%。  
3. **自洽路由已证伪**：头专化越好，`|pred_angle − setting_target|` 对三头都≈0，路由崩（oracle ckpt 仅 38% 一致）。  
4. **joint_phys 有弱正向**：14.43%，非立方 ~3.7%（仍 <5%，但优于 cs_pred 的 2.9%）。

## 可部署配方（目标）

| 组件 | 选择 |
|---|---|
| 输入 | hist + I>5 + matrix6 + 无 2θ shift |
| 头 | `cs_conditional` + `cubic_bravais_split` |
| CS 路由 | 训 oracle / 推 predicted（同 R2） |
| Setting 路由 | 训 oracle / 推 **classifier**（禁止自洽） |
| Loss | SmoothL1 + CE(CS) + CE(setting)；可选 λ≈0.05 `joint_phys` |
| 非立方 | 后续 soft upsample / 专训头（hard-CS ft 在 10k 已伤主指标，慎用） |

## 实验队列（正在跑）

1. ~~100k cs_pred / cubic_split / cubic_split_oracle / joint_phys~~（完成）

## Gate 裁决预期

- **elem**：clf 若 setting 准确率 ≳85%，整体有望 ≥15%（oracle 已 15.93%）。  
- **pull90**：oracle 已 ≤55%；clf 需保持。  
- **ang≤6° / 非立方≥5%**：仍是下一刀（R3.5 / R4），不靠分头单独解决。
