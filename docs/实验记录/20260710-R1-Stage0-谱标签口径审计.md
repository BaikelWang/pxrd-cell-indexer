## 结论

1. **存储峰与 conventional→reduced / primitive / conventional 模拟峰位完全一致**（200 样本：`match_stored_vs_*_2theta = 1.0`，mean abs 2θ / inverse_d2 ≈ 0）。  
   → 输入谱本身没有“错峰”；不是“谱算错了”。
2. **标签是 primitive 六参数**（jsonl 与 `find_primitive()` 一致率 90%；剩余差异来自约化数值/设置惯例）。  
3. **primitive vs conventional 参数差很大**（abs max 均值 ~18°/Å 量级；体积 log-ratio 均值 ~0.40），尤其 cubic/trigonal。  
4. **同一 inverse_d2 直方图特征下**：
   - primitive / jsonl_label：700 过拟合 elem **94% / 95%**
   - conventional：elem **70%**（可学，但更难）  
   → 保持 **primitive 产品口径**；口径差异不是当前 raw 失败的主因（主因仍是喂峰表示）。
