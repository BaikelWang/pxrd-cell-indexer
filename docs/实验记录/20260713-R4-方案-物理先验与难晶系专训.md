# R4 方案：物理先验注入 + 难晶系专训（在 R3 冠军上继续）

> 2026-07-13 · 基线：R3 `cubic_split_clf`（100k strict elem **15.43%**，ang 8.69°，非立方 3.17%，pull90 55.1%）  
> Gate（不变）：elem≥15% · ang≤6° · 非立方≥5% · pull90≤55%  
> 现状：**elem 已过；ang / 非立方 / pull90 未过**（pull90 卡在边界）

## 1. 出发点：把上一轮的物理原理讨论落到实验

Cell indexing 的数学核心是倒易度量张量 \(1/d^2=(hkl)\,\mathbf{G}^*\,(hkl)^\top\)；晶系/Bravais 设定是对 \(\mathbf{G}^*\) 的等式约束。R2/R3 已经把「晶系混训」「立方设定混训」两层结构先验编码进了分头。R4 把还没用上的两类先验继续编码进算法：

1. **确定性的对称固定角**（tet/ortho 的 α=β=90°；hex 的 β=90°，primitive 设定下的经验稳定项，来自 `model/bravais.py` 默认表）——这是**唯一目标**，不是像立方 90/60/109.47 那样要在多个候选里选，所以不会重演 R2 `angle_prior` 的失败（那是在没有分头前，对立方角做 min-over-3-hypotheses，天然会跟真值打架）。
2. **难晶系监督不足**——mono/orth/tet/tric 当前几乎学不到，需要专训 + 温和过采样。

## 2. 零训练先验证：后验 Bravais snap（已跑，无需再等）

在冠军 ckpt 的 raw Top-1 输出上，直接套用 `model/bravais.py` 里已有的 snap 函数（按晶系把预测投影到自己的合法流形上），**不做任何训练**：

| 路由来源 | elem（raw→snap） | ang MAE（raw→snap） | 非立方 elem（raw→snap） |
|---|---|---|---|
| oracle CS | 15.43%→**15.86%** | 8.69°→**8.41°** | 3.17%→**3.42%** |
| predicted CS | 15.43%→**15.86%** | 8.69°→8.58° | 3.17%→3.42% |

**结论**：物理约束方向是对的，但零训练只能薹小甜头（+0.4pp elem，−0.1~0.3° ang）。真正的收益要靠**训练时**就把这个先验编码进 loss，而不是事后补丁。这直接催生了下面的 R4-A。

（脚本：`scripts/eval_manifold_snap.py`，结果：`results/beat_engine/raw_diag/r3/manifold_snap_clf.json`）

## 3. 正在跑的三个 100k 实验

全部基于冠军架构（`histogram + cs_conditional + cubic_bravais_split + setting classifier`），单变量对照：

### R4-A：`manifold_consistency` loss（两个权重点）
- 新 loss 模式：`SmoothL1(matrix6) + λ · manifold_consistency_loss`
- `manifold_consistency_loss`：只对 tet/ortho（α,β→90°）、hex（β→90°）三个晶系的**确定性**固定角做 Huber，cubic/trig/mono/tric 不动（避免歧义打架，避免重复 R2 教训）
- 两个权重：`λ=0.1`、`λ=0.25`，看是否会像 `joint_phys` 一样过重后伤主指标
- 配置：`configs/scale_100k_r4_manifold_w01.yaml` / `_w025.yaml`

### R4-B：难晶系专训 round 2（warm-start 冠军）
- `warm_start_checkpoint` = 冠军 `cubic_split_clf` ckpt
- `hard_cs_finetune_epoch: 1` → 从第 1 个 epoch 起冻 encoder + cubic CS 头 + cubic 设定头，只训 6 个非立方头
- `hard_cs_upsample: 1.5`，`hard_cs_names: [tetragonal, orthorhombic, monoclinic, triclinic]`（**换成了当前实际最差的四个晶系**，不是旧的 hex/trig/mono/tric——hex/trig 现在反而是非立方里最好的）
- 低 LR（3e-4）、短程（10 epoch）避免像 10k 阶段那样把 encoder 训崩
- 配置：`configs/scale_100k_r4_hardcs_ft2.yaml`

> 为什么不先在 10k 上筛：R3 已经证明 **10k 与 100k 方向会反转**（cubic_split/joint_phys 在 10k 全部跑输 cs_pred，在 100k 反超）；10k 筛选在这个任务上不可靠，直接上 100k。

## 4. 执行与产出

- 三个实验串行跑在同一张卡上（`scripts/run_r4_100k_chain.sh`），跑完自动汇总+诊断到 `results/beat_engine/raw_diag/r4/g2_summary.json`。
- 验收表（对每个变体）：elem / ang / 非立方 / pull90 / 分晶系 elem，和冠军逐项对比。

## 5. 决策规则（跑完后怎么选）

| 结果 | 动作 |
|---|---|
| manifold_consistency 某权重 elem 不降、ang 或非立方明显涨 | 采纳，成为新基线 |
| manifold_consistency 两个权重都伤 elem（重演 joint_phys/angle_prior 教训） | 放弃，只保留**推理时**后验 snap（免费的 +0.4pp） |
| hardcs_ft2 非立方明显涨、elem 不崩 | 采纳；否则说明「专训头」这条路线在当前数据量下已到头，转向数据/组成侧 |
| 两者都有正向 | 叠加：在 hardcs_ft2 基础上再套 manifold_consistency |
| 都无效 | 说明当前非立方瓶颈是**数据可辨识性**问题，不是训练协议问题；下一步应看数据构成（难晶系样本质量/数量）或多假设输出，而不是继续调 loss |

## 6. 之后（Gate 仍未过时）的方向，按优先级

1. **多假设输出**（非立方不出单点，出 K 个候选交给后续 FOM）——欠定问题更合理的表述方式，是 Top-K/FOM 阶段本来就要做的事，可以提前在 raw 阶段验证收益。
2. **组成/化学式条件**（如果数据里有）——直接降低 mono/tric 的判定歧义。
3. 输入侧仅做小步消融（峰选择、多尺度 Q），不重开 R1。
4. 若以上都到头：诚实结论是「PXRD-only 单点回归对低对称欠定」，转向排序侯选（回到经典 indexing 框架，网络只做候选生成/打分辅助）。

## 7. 明确不做

- 再拆 cubic（已到 oracle 上界的 96% 左右）。
- 自洽路由、无分类器的设定选择（已证伪）。
- 无 Gate 前的 500k / Top-K / FOM 全量投入。

## 8. 100k 结果与裁决（2026-07-13）

| Run | elem | ang | 非立方 | pull90 | 相对冠军 |
|---|---:|---:|---:|---:|---|
| **cubic_split_clf（冠军）** | **15.43%** | 8.69° | **3.17%** | **55.1%** | — |
| manifold_w01 (λ=0.1) | 14.93% | **8.57°** | 2.67% | 60.2% | elem↓ pull90↑ 否决 |
| manifold_w025 (λ=0.25) | 15.21% | 8.78° | 3.00% | 59.2% | 全面不优 否决 |
| hardcs_ft2（修 warm-start 后） | 14.86% | 8.63° | 2.83% | **54.7%** | elem↓ 非立方↓ 否决 |

**首轮 hardcs_ft2 踩坑**：`trainer.py` 对 histogram 误设 `skip=("encoder.",)`，warm-start 只加载了头、encoder 随机初始化 → elem=0%。已改为同架构全量加载（`skip=()`），重跑后恢复到 ~14.9%。

**裁决**：三个 R4 变体均未超过冠军。保持 `cubic_split_clf` 为部署基线。零训练后验 Bravais snap（+0.4pp）可作为推理免费加成。进入 R5：严口径 Top-K/FOM 诊断。
