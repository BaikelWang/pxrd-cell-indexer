## 设定

| 模式 | 候选生成 |
|---|---|
| **single** | `build_top_k_candidates` + `bravais_set=extended` + `max_log_volume_ratio=log2` + FOM `ref_volume` |
| **search** | `build_manifold_search_candidates`：Bravais 种子 → 晶系流形上 soft-Q L-BFGS 精修 → 体积护栏 + FOM `ref_volume` |

备注：cubic 上 search 的 FOM 从 84.5% 掉到 54.5%——流形精修会漂到错误 Bravais setting，峰拟合仍过但 elementwise 失败。

## 裁决

1. **否决 R6-B 作为 Top-K 主实现**：search 池召回未超过 single 的 17.93%，远未达到方案通过线（≫16.5%，非立方≥10%）。
2. **根因对齐方案预判**：② 初值落在错误盆地时，局部搜索逃不出欠定谷；多晶系精修反而引入错误 setting（伤 cubic FOM）。
3. **附带发现**：在 R6-C 护栏下，仅 `extended` Bravais + 体积护栏的 **single** 已把 pool 抬到 17.93%、非立方 pool 5.67%（相对 R5 的 16.50% / 4.17%）。③ 的增量来自护栏与 extended snap，不是局部搜索。
4. **下一步**：按方案 → **R6-A 峰一致性 loss** 提升②初值；③ 保持 single+护栏，不固化 manifold search。
