# 2026-07-08 — FOM 候选重排序（McMaille 式 de Wolff M(N)）

> **目的**：在 Top-20 候选池几乎必含真解（~99%）的前提下，用观测峰表对候选做 McMaille 式拟合优度重排序，将真解从 rank 3–4 顶到 Top-1，拉高 `top1_lattice_match_rate`。
> **Checkpoint**：`scale_100k_no_cs_matrix6_seed42`（matrix6 最终表示，无需重训）
> **配置**：[`configs/scale_100k_no_cs_matrix6.yaml`](../../configs/scale_100k_no_cs_matrix6.yaml)

---

## 1. 背景

| 指标 | Bravais confidence 排序（基线） | FOM 重排序 |
|---|---:|---:|
| top1_lattice_match_rate | **40.3%** | **54.1%** (+13.8pp) |
| topk_lattice_match_rate | 99.1% | 99.1%（不变） |
| lattice_in_pool | 99.1% | 99.1%（不变） |

基线瓶颈：Bravais snap 的 `confidence = 1/(1+score)` 只衡量「snap 前后几何偏离」，**从未使用观测 2θ 峰表**，真解常在池内 rank 3–4。

---

## 2. 实现

### 新增模块

[`src/pxrd_cell_indexing/model/fom.py`](../../src/pxrd_cell_indexing/model/fom.py)：

| 函数 | 作用 |
|---|---|
| `theoretical_two_theta` | 由候选 (a,b,c,α,β,γ) 枚举 hkl，计算理论 2θ 峰位 |
| `de_wolff_fom` | 计算 de Wolff M(N) = Q_N / (2·N·mean\|ΔQ\|) |
| `rerank_candidates_by_fom` | 对 Top-K 候选池纯 FOM 重排序 |
| `slice_observed_two_theta` | 从 BertModel flatten batch 切出单样本观测峰 |

### FOM 公式与简化说明

- **λ**：固定 Cu Kα 1.54056 Å（与训练 LMDB 模拟谱一致）
- **观测线**：取按 2θ 升序的前 N=min(20, 峰数) 条线
- **Q 匹配**：greedy one-to-one（每条理论线最多用一次）；超出相对容差 `q_match_rtol=0.08` 的匹配记 `unmatched_delta=0.5` 惩罚
- **排序键**（McMaille Occam's razor，升序 = 更优）：
  1. `-n_matched`（最大化命中线数）
  2. `volume`（最小晶胞体积）
  3. `mean_delta`（最小平均 \|ΔQ\|）

> **与出版物严格 de Wolff / McMaille M20 的差异**：本实现为**排序专用简化版**——不做强度加权、不做严格线占用唯一性证明、不用完整 Monte Carlo 局部微调；额外加入**体积最小**二级键以抑制倍胞 scale 变体靠密集反射网格「刷高」FOM 的问题（初版纯 M(N) 排序曾跌至 0.8%，根因即 scale 变体 ΔQ 更小）。

### 接入点

- `LatticeCandidate.fom_score`：可选字段，记录 de Wolff M 值（诊断用）
- `scripts/eval_valid.py`、`scripts/decompose_joint.py`：`--rerank {none,fom}`（D35 后默认 **`fom`**；`none` 用于消融）

## 4. 解读

1. **假设验证成功**：Top-20 召回 ~99% 不变，Top-1 从 40%→54%，说明瓶颈确实是**排序**而非候选生成或模型回归精度。
2. **+13.8pp 显著**：无需重训，纯后处理即可拿到最大单次提升（超过 Decision A/B 全部 ablation 之和）。
3. **仍低于 McMaille ~76%**：差距来自 (a) 模型回归误差导致池内仍有 ~1% 漏检；(b) 排序后仍有 ~45% 池内样本未升到 rank 1；(c) 简化 FOM 与理想峰表假设。后续训练优化（用户第 3 点）可进一步缩小 (a)(b)。
4. **推荐默认开启**：推理/评测链路默认 `--rerank fom`；训练阶段仍用 Bravais confidence 生成池，FOM 仅作后处理。
