# R6 方案：从「回归范式」补足到「回归 + 峰拟合搜索范式」

> 2026-07-13 · 作者：indexing 攻关  
> 基线冠军：`scale_100k_r3_cubic_split_clf_seed42`（严口径 raw Top-1 elem **15.43%**）  
> Gate（不变）：elem≥15% · ang≤6° · 非立方≥5% · pull90≤55%（严口径 = ltol 0.05 / atol 3° / elementwise）  
> 对标对象：McMaille ~66% / JADE ~68%（仅峰+λ）；**不与** RealPXRD Without-L 67%（喂 formula）比

---

## 0. 一句话方案

前几轮（R2–R5）已把 **模块②（原始回归）** 的高对称打到天花板（cubic 89%），低对称（mono 0%、非立方 3%）是 **PXRD-only 单点回归的物理欠定**，继续调 ②/loss/输入已到平台。**R6 把火力从「② 单点回归」转移到「③ 候选搜索 + ④ 峰拟合打分」**，即补上 McMaille 拿到 66% 的那部分范式差距，同时给 ② 加峰几何约束作为保底。

---

## 1. 五模块现状与问题定位

产品 pipeline（对齐 `docs/01-design.md`）：

```
① PXRD输入/预处理 → ② 原始晶格回归 → ③ Bravais/Top-K候选生成 → ④ FOM打分重排 → ⑤ Top-1输出
```

| 模块 | 现状 | 诊断 | R6 动作 |
|---|---|---|---|
| ① 输入 | hist(1/d²)+I>5 | R1/R5-B 已到顶 | **冻结** |
| ② 回归 | cubic 89% / 非立方 3% / ang 8.7° | 低对称欠定，单点回归到平台 | 加峰一致性 loss（保底），定位降为「出初值」 |
| ③ Top-K | 池召回 16.5%（+1pp）；多锚点更差 | **只在②预测点邻域 Bravais snap，非搜索** | **升级为「NN初值 + 多晶系局部搜索 + R因子」（主杠杆）** |
| ④ FOM | 严口径 elementwise **=0%** | 偏好半胞/超胞 | 修：体积护栏 + 惩罚非原胞 + 限尺度变体 |
| ⑤ Top-1 | 当前不如 raw | 被③④拖累 | ③④修好后自然翻转 |

**核心判断**：McMaille 的力量在 ③④（多晶系参数空间搜索 + 峰位 R 因子打分）；我们把宝押在 ② 单点回归，③④ 退化成窄邻域装饰。R6 的重心是 ③④。

## 2. R6-A：模块② 峰一致性损失（可微，保底改造）

### 2.1 动机
当前 loss 是 `SmoothL1(matrix6)`，优化「六个数字像标签」，与 indexing 的定义（预测晶胞算出的理论峰能否对上观测峰）不对齐。加一个**可微峰一致性项**，直接对齐 Bragg 几何。

### 2.2 数学
预测 matrix6 →（`geometry.lattice_params_to_matrix`，torch 可微）3×3 矩阵 A → 倒易度量张量：
\[
G^* = (A^{-1})(A^{-1})^\top,\qquad \frac{1}{d_{hkl}^2} = (h,k,l)\,G^*\,(h,k,l)^\top
\]
- 枚举一组**固定低阶** \(hkl\)（如 \(|h|,|k|,|l|\le 4\)，去重、去 000），得理论 \(q^2_{\text{theory}}\)（一组，随晶胞可微变化）。
- 观测峰：由观测 2θ 经 `two_theta_to_q` 得 \(q_{\text{obs}}\)，取 \(q^2_{\text{obs}}\)。
- **soft Chamfer（对每个观测峰取最近理论峰）**：
\[
\mathcal{L}_{\text{peak}} = \frac{1}{N}\sum_{i=1}^{N} \min_{hkl}\big(q^2_{\text{obs},i} - q^2_{hkl}\big)^2
\]
只对「观测→理论」单向匹配（避免惩罚系统消光缺失的理论峰），可微，量纲需缩放到与 SmoothL1(~0.2) 同量级。

### 2.3 实现
- 新增 `losses.py::peak_consistency_loss(pred_matrix6, obs_two_theta, peak_num, wavelength)`，纯 torch（新建 torch 版 `reciprocal_metric_tensor`；hkl 网格用 buffer 缓存）。
- 新 `LossMode = "peak_consistency"`：`total = SmoothL1 + λ · peak_consistency`（+ 现有 CE(CS) + CE(setting)）。
- 训练时 batch 需带 `pxrd_x/peak_num`（trainer 已有）。
- 权重 `peak_consistency_weight`：先扫 λ ∈ {0.05, 0.15, 0.3}。

### 2.4 实验阶梯（严格执行过拟合闸门）
| 步 | 数据 | 通过线 | 说明 |
|---|---|---|---|
| A-P0 | 700 过拟合 | train elem ≥ 80% 且不劣于纯回归 | 先证明可学，不崩 |
| A-10k | 10k | 不显著低于 cs_pred 5.7% | 注意 10k↔100k 会反转（R3 教训），10k 只筛「崩不崩」 |
| A-100k | 100k | elem ≥ 15.4% 且 ang↓ 或 非立方↑ | 与冠军逐项对比 |

### 2.5 裁决
- ang 明显下降（向 6° 靠）或非立方上升，且 elem 不降 → 采纳。
- 重演 joint_phys/angle_prior（伤 elem 或 pull90）→ 否决，② 保持 baseline，把重心完全交给 ③。

## 3. R6-B：模块③ NN 初值 + 多晶系局部搜索（主杠杆）

### 3.1 动机
McMaille = 六晶系各自 MC 搜参数 + Bragg 峰拟合 R 因子排序。我们已有：
- `refine.py`：L-BFGS 局部精修 + `soft_q_match_loss`（scipy），**但当前只在推理末端对少量候选做微调**。
- `fom.py`：`theoretical_two_theta` / `de_wolff_fom` / `two_theta_to_q`。

R6-B 把 ③ 从「单锚点 Bravais snap + 固定倍胞」升级为「**以 ② 预测为初值的多起点、多晶系局部搜索**」，判据换成峰拟合 R 因子。

### 3.2 设计
对每个样本：
1. **多起点**：取 ② 的 `lattice_norm`（路由头）+ 关键 Bravais snap（cubic P/F/I、tet、hex 等）作为**初值集合**（每个初值代表一个晶系假设）。
2. **局部搜索**：对每个初值用 `refine.py` 的 L-BFGS（或加轻量 MC 扰动重启）以 `soft_q_match_loss` 为目标做局部精修，**约束在该晶系流形上**（如 tet 固定 α=β=γ=90°、a=b；只优化自由参数），避免漂移到半胞。
3. **合并**：所有精修结果 → 去重 → 候选池，交给 ④。

### 3.3 实现
- 扩 `refine.py`：新增「按晶系约束的参数化」（自由参数子集 + 投影回合法流形），复用现有 `soft_q_match_loss`。
- 新增 `scripts/diag_search_topk.py`：对冠军 ckpt，对比 `single`（现状 Bravais snap）vs `search`（多起点局部搜索）的**严口径 pool 召回 + elementwise**，按晶系拆分。**纯推理、零训练、当天可出结果。**
- 关键护栏：搜索目标里加**体积/原胞惩罚**，禁止收敛到半胞/超胞（与 R6-C 一致）。

### 3.4 实验与裁决
- 先零训练在冠军 ckpt 上跑 `single` vs `search`。
- **通过线**：`search` 的 pool elementwise 召回显著 > `single` 的 16.5%（目标 ≥ 30%，尤其非立方 ≥ 10%）。
- 若显著提升 → ③ 成为新的 Top-K 主实现；否则说明 ② 初值太差，局部搜索也逃不出欠定盆地 → 回到 R6-A 提初值质量或转 R6-D。

## 4. R6-C：模块④ 修 FOM 半胞偏好

### 4.1 问题
R5-Diag：FOM 严口径 elementwise **=0%**——FOM 按峰拟合会奖励半胞/超胞（峰能对上但晶胞是真值的 ½ 或 2 倍，mapping 命中、elementwise 失败）。

### 4.2 改法（推理侧，零训练，可与 R6-B 同时验证）
1. **体积护栏**：候选池对 ② 预测体积做 `max_log_volume_ratio` 过滤（`topk.py` 已有 `max_log_volume_ratio_vs_base`，默认关闭 → 打开并调）。
2. **原胞优先打分**：FOM 分里对「更小体积但峰数相同」的候选加惩罚（防止选到超胞），或对倍胞变体降权。
3. **限制/关闭尺度变体**：R5-A 已发现 `all7_noscale` 的 elementwise 略好；在 ④ 打分前对尺度变体强约束。

### 4.3 实验与裁决
- 复用 `scripts/diag_strict_topk_fom.py`，加 `--pool-max-log-volume-ratio` 扫描 + FOM 变体。
- **通过线**：FOM Top-1 elementwise 从 0% 提到接近 pool 召回（即排序不再把真解排在半胞之后）。

## 5. R6-D（兜底 / 需产品决策）：口径或条件

仅在 R6-A/B/C 都无法把严口径 elementwise 推过 Gate 时启动：
1. **改评测口径**：产品接受 volume-guarded / mapping（现有 FOM 池召回已 ~25%），把「严格 elementwise」降为诊断指标。
2. **引入组成条件**：LMDB 有 `p_atom_type` 但管线未暴露；若产品允许「峰+组成」输入（偏离纯 indexing 契约），可大幅压低低对称歧义。需 PM 决策。
3. **定位为候选生成器**：NN 出 Top-K 初值，与传统 Mc/JADE 引擎联合，不以单点 elementwise 为唯一 KPI。

