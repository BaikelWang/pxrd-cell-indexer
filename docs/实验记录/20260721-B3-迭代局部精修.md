# B3 实验报告：B1+B2 之上的迭代局部精修（2026-07-21）

> 承接 `docs/开发日志/20260715-CellIndexing-可执行优化方案v3.md` §13、[B1-S1](20260721-B1-S1-valid1400高对称验证.md)、[B2](20260721-B2-候选排序对比.md) 实验报告。
> 前提（v3 §13）：只有 B1 recall 和 B2 ranking 已通过 Gate 才进入 B3——本轮两者均已通过，具备条件。

---

## 1. 一句话结论

**B3 Gate 未通过（干净的空结果，不是失败）：迭代局部精修对 ranked Top-1 几乎没有影响**（整体 57.5%→58.1%，仅 +0.6pp，远低于 Gate 要求的 +2pp），且**候选自身的角度参数在精修前后完全没有变化**（angle MAE 5.024°→5.024°，逐晶系同样精确不变）。原因很直接：当前 valid1400 是**无噪声模拟数据**，q-search 用于精确解的低角峰本身就是理论峰的精确值，最小二乘用全部匹配峰重新拟合 G* 时没有任何残差可供优化——候选早已在浮点精度极限上"精确"，精修没有可以挤出的信号。唯一的微小变化（+0.6pp 主要来自 tetragonal +2.5pp）来自候选池内部**其他候选**的匹配数在重拟合后轻微变化导致的重排序，不是目标候选自身被"修好"。**决策：不采用 B3；继续用 B1（独立 q-search）+ B2（legacy FOM + NN 体积先验）作为生产候选流水线；B3 的价值留给未来引入真实谱噪声（S3-A4 类扰动）之后再评估。**

---

## 2. 方法

- 复用 B1-S1/B2 同一套固定数据（cubic/tetragonal/hexagonal/trigonal 各 40 valid1400 真实样本，pool_budget=20，oracle CS 路由）。
- 新增 `src/pxrd_cell_indexing/search/refine.py`：对给定候选，把**全部**观测峰重新匹配到当前 G* 下的理论 hkl 网格（贪心 1:1，同 de Wolff 匹配逻辑），再用**全部**匹配到的 (hkl, q_obs) 对做（超定）线性最小二乘重新解该晶系自由度的 G* 分量——注意这是**精确线性解**，不是梯度下降；关键动机是初始 q-search 只用了 k 个（晶系自由度）低角峰做精确解，精修让"后来被 de Wolff 校验认出来的"其余匹配峰也参与进来收紧估计。每步保持 SPD、matched 数不下降才接受，否则原样保留（v3 §13 的"停止 refiner"要求）。
- 流程：B1 池 → B2 排序（legacy FOM + NN 体积先验，B2 的最优方法）→ 取 Top-5（M=5）→ 对每个候选独立跑 0/1/3 步精修 → 用同一 FOM 对精修后的 Top-5 重新排序 → 检查新 ranked Top-1 是否命中。
- 脚本：`scripts/run_b3_refine_valid1400.py`；原始结果：`results/beat_engine/b1_search/b3_refine_valid1400.json`。

## 4. 为什么精修没有效果（诚实的负结果）

q-search 的初始精确解，本身就是用真实晶系自由度个低角峰、解一个**精确**（非最小二乘）的线性方程组得到的——如果 hkl 分配正确，且观测峰位置是无噪声理论值（valid1400 目前就是这种数据，2θ 直接来自 pymatgen `XRDCalculator` 的确定性计算，没有实验误差/展宽/零点漂移），那么这个解在浮点精度内就是**精确解**，不存在可供最小二乘"收紧"的残差。本轮精修用全部匹配峰重新拟合，理论上能纠正的场景是：初始 k 峰解因某种原因（如 k 峰本身在某个方向上病态、或后续认出的匹配峰其实略微不一致）不是最优解——但当前数据里这种场景基本不存在，所以 0/1/3 步的候选参数完全没有变化（angle MAE 逐 bit 相同）。整体 ranked Top-1 的微小 +0.6pp 波动，来源是 Top-5 中**其他**候选（不是被检验的那个）在重拟合后匹配数发生了 ±1 的变化，从而在重排序时反超——这是排序噪声，不是精修本身的收益。

**这个结论对当前"生产用什么"的决策没有影响**（B1+B2 已经通过各自的 Gate，足够支撑），但对**未来**有指导意义：如果后续要在真实实验谱（有零点漂移、峰位抖动、展宽等，参见 S3 的 robust perturbation 组件）上部署这套 pipeline，届时观测峰位不再是理论精确值，B3 式的最小二乘精修就会有真实残差可收——那时应该重新跑一次同样的 B3 Gate 测试，而不是假设本轮的空结果一直成立。

---

## 5. 决策

- **不采用 B3**：当前（无噪声）数据下没有可观测收益，不satisfy Gate。
- **生产候选流水线维持 B1（独立 q-search）+ B2（legacy FOM + NN 体积先验）**，不叠加精修步骤。
- `search/refine.py` 代码保留（已测试、行为符合预期），作为**未来真实谱噪声场景**下的现成工具，届时优先复测本报告的 Gate，而非直接假定其有效或无效。
