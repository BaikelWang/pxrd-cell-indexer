# 实验 B1 — 严口径对齐损失（B1a hinge / B1b angle-heavy）

> **日期**：2026-07-09  
> **状态**：✅ 完成（**两项均未达 Gate-2；相对 B4 为负向或崩塌**）  
> **总纲**：[`20260709-完胜引擎攻关方案v1.md`](../开发日志/20260709-完胜引擎攻关方案v1.md) §8.2  
> **单因素**：在 B4 选模 + warm_start(B4 best) 上只改 loss mode  
> **对照**：B4 valid strict elem **5.6%** / MP100 strict raw **4%**

---

## 1. 假设

宽松 SmoothL1 不感知 `0.05/3°` 边界；用 hinge 或角度重加权可把梯度压到严口径附近，抬高 `strict_raw_*_elementwise`。

**成功标准**（方案）：严口径 fom ≥30% 且较段1最优 ≥+10pp。

---

## 2. 实现

| 项 | 说明 |
|---|---|
| `LossMode` | `strict_hinge`（B1a）、`angle_heavy`（B1b） |
| hinge | `relu(rel_len - 0.05)` + `relu(|Δθ| - 3°)`，与 norm SmoothL1 相加 |
| angle_heavy | 物理 length+angle Huber，`angle_weight` 至少 5×，**替换** norm regression |
| config | `configs/scale_100k_b1a_strict_hinge.yaml`、`..._b1b_angle_heavy.yaml` |
| 测试 | `tests/test_losses.py`（hinge 容差内外、angle_heavy 加权）10 passed |

训练：100k，8 epoch，`best_metric=strict_raw_top1_elementwise_rate`，warm_start = B4 best。

## 4. B1b — angle_heavy

### 训练曲线（valid）

| ep | loose raw | strict map | **strict elem** | angle_mae |
|---|---:|---:|---:|---:|
| **1** | 28.1% | 0.14% | **0.14%** | 9.31° |
| 2 | 24.4% | 0% | 0% | 9.18° |
| 3 | 22.0% | 0% | 0% | 9.10° |
| 4 | 24.1% | 0.14% | 0% | 9.01° |

early-stop@ep4（patience=3），best 停在 ep1 的 **0.14%** — 相对 B4 **崩塌**。  
根因：`angle_heavy` **完全丢掉** matrix6 空间 SmoothL1，物理角加权主导后破坏已学到的表示。

---

## 5. 结论

1. **B1a 假设部分否定**：hinge 略压 `angle_mae`（9.75→9.21），但 **严口径 elementwise 变差**（5.6→2.8）；loose 略升说明优化在「宽松盆地」里打转，未进入 0.05/3°。  
2. **B1b 明确失败**：替换式物理 loss 不可用；后续若再试 angle 加权，必须 **与 regression 相加**（类似 hinge/combined），不可替换。  
3. **Gate-2 未过**；loss 微调无法单独把 raw 从 ~5% 抬到 30%。  
4. **保留 B4 ckpt 作为段2 默认基线**；B1a/B1b 不进入后续 warm_start 链。
