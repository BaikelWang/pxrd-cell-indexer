# R3 方案：Bravais 设定分头 + 联合物理损失 + 难晶系专训

> **日期**：2026-07-10  
> **冻结基线**：R2 `histogram + cs_conditional + cs_pred`（100k strict elem 13.57%）  
> **Gate（同 R2）**：elem≥15%、ang≤6°、非立方≥5%、pull90≤55%

## 问题（一句话）

条件化抬了 cubic 天花板，但 **同一 cubic 头混训 90°/60°/109.47°**，且非立方仍是 length∧angle 联合失败；R2 的「往 90° 推」先验已证伪。

## 三板斧

### A. Cubic Bravais 设定分头（主杠杆）

- 数据：cubic 中 F+I 约占 79%，与 P 的 90° 混在同一头会学成均值角。  
- 做法：cubic 再拆 3 个头，对应角设定 `{90, 60, 109.47}`。  
- 训练：用 GT 平均角最近邻选设定（oracle setting）。  
- 推理：三个 cubic 头都跑，选「预测角最接近自身设定」的头（自洽路由）。  
- 非立方仍用原 7 头中对应头。

### B. 加性联合物理损失 `joint_phys`（辅）

- `loss = SmoothL1(matrix6) + λ * (rel_length + Huber(angle))`，目标是 **真值**，不是 90°。  
- λ 小（0.05–0.2）；**禁止**替换主回归（B1b 教训）。  
- 与已失败的 `angle_prior` 不同。

### C. 难晶系专训（辅）

- 阶段 1：正常训满。  
- 阶段 2：冻结 encoder + cubic 相关头，只更新 tet/orth/hex/trig/mono/tric 头，并对难晶系过采样 batch（温和，非 B2 全局×2 复制）。

## 实验阶梯

1. **700**：cs_pred × {baseline, cubic_split, joint_phys, both}  
2. **10k**：同上 + hard-CS finetune 消融  
3. **100k**：冠军设定 vs R2 cs_pred 对照，严口径 + diagnose

## 停止项

未过 Gate 不做 500k / Top-K / FOM。

## 100k 关键发现（进行中）

| 设定 | strict elem | 备注 |
|---|---:|---|
| R2/R3 cs_pred | 13.57% | 冻结基线 |
| cubic_split + **predicted** 自洽路由 | 10.64% | 路由失败，比基线差 |
| cubic_split + **oracle** setting | **15.93%** | 过 elem≥15%；cubic 94%；pull90 54.6% |
| joint_phys | 14.43% | 非立方最好 ~3.7% |

**结论**：分头本身有效；可部署瓶颈是 **setting 路由**，不是分头容量。

### 自洽路由为何失败（探针）

在 `cubic_split` / `cubic_split_oracle` 的 valid cubic（n=200）上：

| ckpt | 路由与 GT 一致 | cubic elem（oracle 头） | cubic elem（自洽） |
|---|---:|---:|---:|
| predicted 训 | 66.5% | 90.5% | 63.5% |
| oracle 训 | **38.0%** | **94.5%** | 36.0% |

- 按 L1 选最优头 **100% = GT setting** → 三个头已正确专化。  
- 头越好，自洽越差：每个头的预测角都贴近自身目标，`|pred−target|≈0` 对所有头成立，路由信号消失。  
- **因此必须用 embedding 上的 setting classifier**（训时 oracle 选头 + CE；推时 argmax），不能靠自洽角距离。
