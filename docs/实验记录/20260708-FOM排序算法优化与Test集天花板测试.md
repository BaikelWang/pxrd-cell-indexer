> **目的**：在不重训的前提下，把 FOM 重排序推到能力上限，并在从未用于任何决策的 hold-out test 集上一次性报告天花板数字，为「是否继续优化 AI 回归精度」提供依据。
> **Checkpoint**：`scale_100k_no_cs_matrix6_seed42`（matrix6，无重训）
> **调参集**：valid1400（`pxrd_241113_valid.lmdb`）
> **天花板集**：test1400（`pxrd_241113_test.lmdb`，25552 条全量中分层抽 1400，与 valid 同方法学）

## 1. 根因与修复

### 1.1 波长系统性偏差（已修复）

| 项 | v1（D34） | v2（本实验） |
|---|---|---|
| `DEFAULT_WAVELENGTH_ANGSTROM` | 1.54056（Cu Kα1） | **1.54184**（pymatgen `XRDCalculator()` 默认 Cu Kα 加权平均） |
| 依据 | 误用常见文献值 | [`241113_save_pxrd_data.py`](../../../alex_aflow_oqmd_mp/codes/241113_save_pxrd_data.py) 无参数 `XRDCalculator()` |
| 真解自洽检验 | ΔQ ~2e-4–6e-4 | **ΔQ = 0.0**（机器精度） |

v1 的 8% 相对容差本质上在补偿该波长误差，同时让错误超胞候选也能「凑巧命中」大量弱峰。

### 1.2 容差非单调行为（已排查）

修复波长后，在 valid1400 上 sweep 绝对容差 `q_match_abs_tol`（`rtol=0`）：

| q_match_abs_tol | top1_lattice_match_rate | 解读 |
|---:|---:|---|
| 1e-4 | **5.8%** | 过松：超胞/错误候选靠 spurious 命中刷高 n_matched，真解被压到 rank 2+ |
| 1e-5 | 39.0% | 中间态：部分过滤 spurious，但仍有 float/近简并分配残留 |
| 1e-6 | **85.9%** | 理想棍谱最佳点：拒绝 spurious，又保留真解机器精度匹配 |

**结论**：理想模拟谱应使用**极紧绝对容差**（`1e-6`），而非 v1 的 8% 相对容差。非单调来自「容差过松时错误候选获益更多」而非真解受损。

### 1.3 排序公式对比（valid1400 网格，18 组）

网格：`mode ∈ {heuristic, strict_dewolff, intensity_weighted}` × `collapse_variants ∈ {false, true}` × `q_abs_tol ∈ {1e-4, 1e-5, 1e-6}`

**在最佳容差 1e-6 下，三种 mode 与 collapse 均无显著差异**（均 ~85.9%）。完整 sweep 见 [`results/fom_rerank_sweep_valid1400.json`](../../results/fom_rerank_sweep_valid1400.json)。

| 配置（tol=1e-6） | top1 | top1\|in_pool |
|---|---:|---:|
| heuristic | **85.9%** | 86.6% |
| strict_dewolff | 85.9% | 86.6% |
| intensity_weighted | 85.8% | 86.5% |
| + collapse_variants | 85.9% | 86.6% |

**选定最优配置**（valid1400 最高，与其他并列）：`mode=heuristic`，`collapse_variants=false`，`q_match_abs_tol=1e-6`。

## 4. 天花板解读：排序 vs 召回之间还剩多少？

```
Top-K 召回（候选生成 + 模型回归）  ≈ 99.8%
         ↓  FOM v2 排序后 Top-1     ≈ 87.6%
         ─────────────────────────────────
         不可由排序挽回的差距        ≈ 12.2%（池内真解未升到 rank 1）
         + 0.2%（真解未入池）
```

| 漏斗环节 | 占比 | 可否靠排序解决 |
|---|---:|---|
| 真解未入 Top-20 | ~0.2% | 否——需改进候选生成/回归精度 |
| 真解在池内但非 rank 1 | ~12.2% | **否——已达排序公式上限** |
| 真解 rank 1 | ~87.6% | — |

**PM 决策依据**：

1. **FOM v2 必须采纳**：相对 v1 再 +31.8pp（54%→86%），相对无 FOM +47.3pp（40%→87%），工程改动小。
2. **排序优化已接近天花板**：在 99% 召回前提下，纯后处理最多再榨出 ~12% 到 Top-1 的「排序不可达」残差；继续堆 FOM 公式收益递减（三种 mode + 去重均无增益）。
3. **下一步主战场是 AI 回归精度**：要把 Top-1 从 ~88% 推向 95%+，需要 (a) 缩小模型回归误差使真解在池内更靠前；(b) 减少超胞变体对真解的「峰表混淆」——这属于 **500k/全量训练 + 候选池策略** 范畴，而非 FOM 微调。

| 判定项 | 结论 |
