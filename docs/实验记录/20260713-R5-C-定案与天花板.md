## 3. 核心结论

**在当前数据与 Gate（elementwise 严口径）下，PXRD-only 单点回归对低对称晶系是欠定问题。**

证据链：
1. **高对称已解开**：立方分头 + setting classifier 把 cubic 推到 ~89%，elem 过 15%。
2. **低对称学不动**：mono≈0%，非立方均值 ~3%；R4 专训 / 物理先验 / 输入加细均抬不动。
3. **候选扩展救不了 elementwise**：单锚 Bravais +1pp；多锚点反而挤掉真解；FOM 在 mapping 口径有用（池召回 ~25%、FOM mapping ~19%），但对 Gate 的 elementwise 给出 0%（尺度变体偏好半胞/超胞）。
4. **宽口径 FOM 天花板不可直接搬过来**：M2.5 的 ~86% 是宽松 mapping 排序；严口径 elementwise 是另一套问题。

## 4. 数据侧需求（当前管线没有）

- LMDB 有 `p_atom_type`（逐位点元素），但 **训练/评测管线未暴露 composition/formula**（`DatasetSample` 仅有 `atom_num`）。
- 若要继续冲低对称：需要向数据方确认是否可提供**组成条件**（化学式 / 元素集合）进入模型，或接受 **mapping/体积护栏** 作为产品口径（而非严格 elementwise）。

## 5. 本阶段推荐交付

1. **部署模型**：`cubic_split_clf`（histogram + CS 条件化 + cubic Bravais 分头 + setting classifier）。
2. **可选后处理**：推理时 Bravais snap（+0.4pp elem）。
3. **不建议**：在同一 Gate 下继续广扫 loss / 输入 / 多锚点；投入产出已到平台。
4. **下一阶段选项**（需产品决策）：
   - 放宽评测到 volume-guarded / mapping，再启用 FOM（并关掉或限制尺度变体）；或
   - 引入组成条件；或
   - 把网络定位为「候选生成器」，与传统 indexing 引擎联合，不以单点 elementwise 为唯一 KPI。
